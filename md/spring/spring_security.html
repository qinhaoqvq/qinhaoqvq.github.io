    # 核心概念

    • Authentication：认证，验证当前访问系统的是不是本系统的用户，并且要确认具体是哪个用户
    • Authorization：授权，经过认证后判断当前用户是否有权限进行某个操作

    # 基本原理

    # 过滤器介绍

    Spring Security 本质是一个过滤器链

    # UsernamePasswordAuthenticationFilter

    简单来说就是负责认证。它会过滤登录的URL，获取前台输入的用户名，密码并封装成UsernamePasswordAuthenticationToken对象，之后再通过AuthenticationManager.authenticate()进行校验

    # ExceptionTranslationFilter

    处理过滤器链中抛出的任何AccessDeniedException 和 AuthenticationException

    # FilterSecurityInterceptor

    简单来说就是负责授权。它是方法级的权限过滤器，基本位于过滤器链最底部

    # 过滤器链的加载流程

    DelegatingFilterProxy -> FilterChainProxy -> 通过 getFilters 加载过滤器

    # 核心接口

    # UserDetailsService

    通过该接口自定义用户信息查询逻辑，一般都是查询数据库，返回对象需要是UserDetails类型

    # PasswordEncoder

    数据加密接口，用于返回用户对象密码的加密

    # 认证授权注解

    通过@EnableGlobalMethodSecurity注解开启认证授权注解

    • @Secured：判断是否拥有指定角色，通过securedEnabled = true启用
    • @PreAuthorize：方法执行前进行权限验证，通过prePostEnabled = true启用
    • @PostAuthorize：方法执行后进行权限验证，通过prePostEnabled = true启用。该注解不经常使用
    • @PreFilter：进入方法前对请求数据进行过滤
    • @PostFilter：权限验证后对响应数据进行过滤

    # 思路分析

    登录

    1. 自定义登录接口
      1. 通过AuthenticationManager调用 authenticate 方法进行认证，认证成功生成JWT
      2. 将用户信息存入 redis 缓存中
    2. 实现UserDetailsService接口
      1. 重写loadUserByUsername方法，通过数据库查询用户信息，并返回 UserDetails 对象

    校验

    1. 定义JWT认证过滤器
      1. 获取并解析 Token，获取用户信息
      2. 从 redis 缓存中获取详细的用户信息
      3. 存入 SecurityContextHolder

    # 跨域问题

    # 后端服务开启允许跨域请求

    通过重写WebMvcConfigurer#addCorsMappings方法可以开启跨域访问

    # Spring Security 开启允许跨域请求

    通过调用httpSecurity.cors()方法可以开启跨域访问

    # 自定义权限校验方法

    **方法定义要求：**返回值需要为 boolean 类型

    通过SpEL表达式进行方法调用，例如，@PreAuthorize("@component('sys:perms:query')")
