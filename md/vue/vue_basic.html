    # 初识 Vue

    • 想让 Vue 工作，就必须创建一个 Vue 实例，并且要传入一个配置对象

    • 容器里的代码被称为「Vue 模板」
    • Vue 实例和容器的关系是一一对应的
    • 实际开发过程中只会有一个 Vue 实例，并且会配合着组件一起使用
    • 中的 xxx 要写 JavaScript 表达式，且 xxx 可以自动读取到 data 中所有属性
    • 一旦 data 中的数据发生改变，那么模板中使用到该数据的地方也会自动更新

    # Object.defineProperty() 方法

    定义对象中新属性或修改原有的属性

    # 基本语法

    Object.defineProperty(obj, prop, descriptor)
- obj: 必填，操作的目标对象
- prop: 必填，需要定义或修改的属性名称
- descriptor：必填，目标属性所拥有的特性

    # 参数 descriptor 说明

    • value：设置属性的值，默认为 undefine
    • writable：设置属性是否可以被修改，默认为 false
    • enumerable：设置属性是否可以被枚举，默认为 false
    • configurable：设置属性是否可以被删除或是否可以再次修改特性参数，默认为 false

    # 👺Vue 中的数据代理

    图片

    • vue 中数据代理的应用：通过 vm 对象来代理 data 对象中属性的操作（读/写）
    • vue 中数据代理的好处：更加方便操作 data 中的数据
    • 基本原理：通过 Object.defineProperty() 把 data 对象中所有属性添加到 vm 中。为每一个添加到 vm 的属性都指定一个 getter/setter 方法，然后再在 getter/setter 内部去操作（读/写）data 中对应的属性

    # 事件处理

    当用户单击某个元素的时候，我们给这个元素注册的事件就会触发，该事件的本质就是一个函数，而该函数的形参接收一个 event 对象

    # 基本用法

    1. 使用 v-on:xxx 或 @xxx 绑定事件，其中 xxx 是事件名称
    2. 事件的回调需要在配置在 methods 对象中，最终会在 vm 对象中
    3. methods 中配置的函数，都是被 Vue 所管理的函数，this 指向的是 vm 或者组件实例对象。定义函数是不要使用箭头函数，否则 this 指向的不是 vm
    4. @click="function" 和 @click="function($event)" 效果是一样的，都能在函数中获取到事件对象，通过 event.target 可以获得触发此事件的元素

    # 事件修饰符

    1. prevent：阻止默认事件（常用）
    2. stop：阻止事件冒泡（常用）
    3. once：事件只触发一次（常用）
    4. capture：使用事件的捕获模式
    5. self：只有 event.target 是当前操作的元素时才触发事件。可以使用该修饰符实现与 prevent 类似效果
    6. passive：事件的默认行为立即执行，无需等待事件回调执行完毕

    # 键盘事件

    1. keydown：只要按下按键就会触发该事件
    2. keyup：当按下按键再松开按键后才会触发事件

    # Vue 中常使用的按键别名

    1. 回车 => enter
    2. 删除 => delete
    3. 退出 => esc
    4. 空格 => space
    5. 换行 => tab（该按键比较特殊，必须配置 keydown 使用）
    6. 上 => up
    7. 下 => down
    8. 左 => left
    9. 右 => right

    # 未提供别名的按键

    Vue 未提供别名的按键，可以使用该按键的原始 key 值（event.key 可以获取）去绑定，但要注意转为 kebab-case（短横线命名）。例如，大小写（CapsLock） => caps-lock

    # 系统修饰符

    ctrl、alt、shift、meta(win)

    1. 配合 keyup 使用：按下系统修饰符的同时，再按下其他键，之后释放其他键，事件才会被触发
    2. 配合 keydown 使用：正常触发事件

    可以使用组合键：@keyup.ctrl.y 表示需要同时按下 ctrl + y 才能触发事件

    # 直接使用 keyCode（不推荐）

    可以直接使用 keyCode（event.keyCode 可以获取） 值进行键盘事件绑定，@keyup.13，13 => enter。但是不推荐以此方式，因为不同的键盘或操作系统 keyCode 值可能不一样

    # 自定义按键别名

    Vue.config.keyCodes.queren = 13

    # 计算属性

    • 定义：要使用的属性 data 中不存在，但是可以通过已有的属性计算得来
    • 原理：底层使用了 Object.defineProperty() 方法提供的 setter 和 getter
    • get 函数什么时候执行？
      • 第一次读取时会被执行
      • 当依赖的属性发生改变时会执行
    • 优点：与 methods 实现方式相比，内部有缓存机制（复用），效率更高
    • 计算属性最终会出现在 vm 对象中，在模板中可以直接读取
    • 如果计算属性允许被修改，那必须重写 setter 方法，并且 set 方法中要同步修改依赖属性的值

    # 基本语法

    data() {
	return {
		firstName: '张',
		lastName: '三',
	}
},
computed: {
	fullName: {
		get() {
			const fn = this.firstName + this.lastName
			return fn
		},
		set(value) {
            fullName = value
          	const array = value.split('-')
          	this.firstName = array[0]	// 修改依赖属性的值
          	this.lastName = array[1]	// 修改依赖属性的值
		}
	}
}

    # 简写方式

    如果设计定义计算属性时，只考虑它可读（getter），不考虑可写（setter）的情况，那么定义的方式就可以简化

    ...
computed:{
  fullName(){
    const fn = this.firstName + this.lastName
    return fn
  }
}

    # 条件渲染

    # v-if

    v-if 指令用于条件性地渲染一块内容，

    # v-show

    v-show 指令用于条件性地渲染一块内容，但是v-show 不支持 <template> 元素，也不支持 v-else

    # template

    用于切换多个元素，可以把 <template> 元素当做不可见的包裹元素，并在上面使用 v-if。最终的渲染结果将不包含 <template> 元素。

    # v-if 和 v-show

    • v-if 它会确保在切换过程中条件块内的事件监听器和子组件适当地被销毁和重建
    • v-show 不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换，即display: none
    • v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好

    # 列表渲染

    v-for 指令基于一个数组来渲染一个列表。v-for 指令需要使用 item in items 形式的特殊语法，其中 items 是源数据数组，而 item 则是被迭代的数组元素的别名

    # 🤖Key 的作用与原理

    # 抛出问题

    # 场景一

    在使用条件渲染时，如果生成的DOM元素内部包含了表单输入，那么当DOM元素再动态渲染时，表单输入可能不会被正确的渲染

    <template v-if="loginType === 'username'">
  <label>Username</label>
  <input placeholder="Enter your username">
</template>
<template v-else>
  <label>Email</label>
  <input placeholder="Enter your email address">
</template>
// 当 input 输入框中有值后，再做动态渲染，input 会被复用，里面的值也存在

    # 场景二

    在使用列表渲染时，如果生成的DOM元素内部包含了表单输入，那么当改变了 items顺序时（头插入、重排序等），表单输入可能不会被正确的渲染

     <ul>
	<template v-for="(item,index) in persons">
		<li>{{item.name}},{{item.age}} <input /></li>
	</template>
</ul>
// 当在 persons 头部插入新的对象时，会改变数组的顺序，会导致 Vue 虚拟 DOM 重新生成

    # 问题解决

    在动态渲染元素时，Vue 会尽可能高效地渲染元素，通常会复用已有元素而不是从头开始渲染，如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序，而是就地更新每个元素，只适用于不依赖子组件状态或临时 DOM 状态 (例如：表单输入值) 的列表渲染输出。通过指定元素的 key 来帮助 Vue 做虚拟 DOM 的对比。

    # 概念

    key是虚拟 DOM 对象的标识，当数据发生变化时，Vue 会根据「新数据」重新生成「新的虚拟DOM」，之后 Vue 会进行「新虚拟DOM」和「旧虚拟DOM」的差异比较

    # 对比规则

    • 在旧虚拟DOM中找到了与新虚拟DOM相同的key
      • 若虚拟DOM中内容没有改变，则复用之前生成的真实DOM
      • 若虚拟DOM中的内容改变了，则根据新的虚拟DOM重新生成真实DOM，替换页面中原有的真实DOM
    • 在旧虚拟DOM中未找到与新虚拟DOM相同的key
      • 根据新的虚拟DOM生成真实DOM，然后渲染到页面

    # 使用 index 作为 key 可能会存在的问题

    • 如果对数据进行逆序添加、逆序删除等重排序操作，会产生没必要的真实DOM更新，效率低
    • 如果对数据进行逆序添加、逆序删除等重排序操作，并且结构中包括表单输入DOM，会产生DOM更新，界面展示有问题

    # 解析流程

    1. Vue 会根据数据生成虚拟 DOM
    2. 然后将虚拟 DOM 转换为真实 DOM，页面展示
    3. 当依赖数据发生变化时，会重新的解析模板
    4. Vue 会根据新的数据重新生成虚拟 DOM
    5. 新虚拟 DOM 与 旧虚拟 DOM 会进行对比算法 diff
    6. 对比算法，会依赖 key
    7. 首先，获取新虚拟 DOM 中第一个元素，并获取 Key值，然后通过key去旧虚拟 DOM 中进行查找
    8. 如果找到了 key相同的元素，然后再依次对比元素中的节点
    9. 如果节点在旧 DOM 元素中不存在或者不相同，则会根据新虚拟 DOM 元素生成真实 DOM 元素
    10. 如果节点在旧 DOM 元素中存在，则会直接复用旧 DOM 已经生成好的真实 DOM 元素。（上述问题，就是出现在这一步，因为对比的是虚拟 DOM，对比结果相同，但是，如果复用真实的 DOM 元素，可能会保存了之前的数据）
    11. 如果未找到 key相同的元素，则根据新虚拟 DOM 生成真实 DOM 元素

    # 👺Vue 深入响应式理解

    # 抛出问题

    由于 JavaScript 的限制，Vue 不能检测数组和对象的变化

    # 场景一

    对于对象来说，Vue 无法检测 property 的添加或移除。由于 Vue 会在初始化实例时对 property 执行 getter/setter 转化，所以 property 必须在 data 对象上存在才能让 Vue 将它转换为响应式的。对于已经创建的实例，Vue 不允许动态添加根级别的响应式 property

    <h1>对象</h1>
<ul>
	<li>姓名: {{person.name}}</li>
	<li>年龄: {{person.age}}</li>
	<li>地址: {{person.address}}</li>
	<li>其实这里还有一个为空的属性 isLeader: {{person.isLeader}}</li>
</ul>
person: {
	name: 'admin',
	age: 22,
	address: 'Shanghai'
	// 定义 person 时并没有定义 isLeader 属性，那么在获取该属性值时为 undefined ，Vue 页面不会展示 undefined
}
    1. 通过 person.isLeader = 'value'赋值，person 对象实际已经添加了 isLeader 属性，但是页面没有显示出该值
    2. 因为 isLeader 不是响应式的 property。通过该方式添加属性，data中不会为该属性生成 getter/setter 方法，那么在修改该值时也就不会被检测到，从而页面也不会重新渲染加载该值

    # 场景二

    对于数组来说，Vue 无法检测以下数组的变动：

    1. 当你利用索引直接设置一个数组项时，例如：vm.items[indexOfItem] = newValue
    2. 当你修改数组的长度时，例如：vm.items.length = newLength
    <h1>数组</h1>
<ul>
  <li v-for="(person,index) in persons" :key="index">person: {{person}}</li>
</ul>
persons: ['zhangsan', 'lisi', 'wangwu']
    1. 通过 persons[0]="newValue"赋值，persons 下标为0的值实际已经被修改，但是页面没有显示出最新值
    2. Vue 在加载数组数据时，只为数组本身维护了getter/setter，但并未数组中的每一个元素添加getter/setter，所以当数组中元素发生变化时，无法被检测到

    # 问题解决

    1. Vue.set（vm.someObject, 'property', 'value'）
    2. vm.$set（vm.someObject, 'property', 'value'）
    3. Vue 将被侦听的数组的变更方法进行了包裹，所以它们也将会触发视图更新
      1. push()
      2. pop()
      3. shift()
      4. unshift()
      5. splice()
      6. sort()
      7. reverse()
    4. 替换数组的方式
    5. Vue.set 和 vm.$set 不能给 vm 或 vm._data 添加属性

    # 概念

    当你把一个普通的 JavaScript 对象传入 Vue 实例作为 data 选项，Vue 将遍历此对象所有的 property，并使用 Object.defineProperty (opens new window) 把这些 property 全部转为 getter/setter。每个组件实例都对应一个 watcher 实例，它会在组件渲染的过程中把"依赖"的数据 property 记录。之后当依赖项的 setter 触发时，会通知 watcher，从而使它关联的组件重新渲染。参考资料 (opens new window)

    # 原理验证

    let data = {
	name: 'admin',
	age: 18
}

// 1. 定义监视对象
function Observer(data) {
  let keys = Object.keys(data)
  keys.forEach((key) => {
    Object.defineProperty(this, key, {
      get() {
        console.log('getter...')
        return data[key]
      },
      set(val) {
        console.log('setter...')
        data[key] = val
      }
    })
  })
}
// 2. 生成vm对象, vm._data = data = new Observer(data)
const vm = {}
vm._data = data = new Observer(data)

    # 表单输入绑定

    可以用 v-model 指令在表单 <input>、<textarea> 及 <select> 元素上创建双向数据绑定

    # 基本语法

    • CheckBox，如果是单个复选框，则绑定到布尔值。如果是多个复选框，则绑定到数组，数组值为 value 值

    # 修饰符

    • .lazy：在默认情况下，v-model 在每次 input 事件触发后将输入框的值与数据进行同步，可以通过 lazy 修饰符，从而转为在 change 事件之后进行同步
    • .number：自动将用户的输入值转为数值类型
    • .trim：自动过滤用户输入的首尾空白字符

    # Vue 过滤器

    Vue.js 允许你自定义过滤器，可被用于一些常见的文本格式化。过滤器可以用在两个地方：双花括号插值和 v-bind 表达式 (后者从 2.1.0+ 开始支持)。过滤器应该被添加在 JavaScript 表达式的尾部，由_管道_符号指示

    # 基本语法

    # 调用

    <!-- 在插值语法中 -->
{{ message | capitalize }}
<!-- 在 `v-bind` 中 -->
<div v-bind:id="rawId | formatId"></div>

    # 定义

    1. 局部过滤器：filters:{capitalize:function(value,其他参数){}}
    2. 全局过滤器：Vue.filter('capitalize', function (value,其他参数) {})

    # 内置指令

    # 基础指令

    1. v-bind：单向绑定表达式，简写 :xxx
    2. v-model：双向数据绑定
    3. v-on：绑定监听时间，简写 @
    4. v-for：遍历数组/对象/字符串
    5. v-if、v-else-if、v-else：条件渲染（动态控制DOM元素是否存在）
    6. v-show：条件渲染（动态控制DOM元素是否显示）
    7. v-text：内容会按文本插入指定节点
    8. v-html：内容按普通 HTML 插入 - 不会作为 Vue 模板进行编译。在网站上动态渲染任意 HTML 是非常危险的，因为容易导致 XSS 攻击
    9. v-clock：不需要表达式，该指令保持在元素上直到关联实例结束编译。和 CSS 规则如 [v-cloak] { display: none } 一起用时，这个指令可以隐藏未编译的 标签直到实例准备完毕（可用于页面加载渲染慢优化）
    10. v-pre：不需要表达式，跳过这个元素和它的子元素的编译过程
    11. v-once：不需要表达式，只渲染元素和组件一次。随后的重新渲染，元素/组件及其所有的子节点将被视为静态内容并跳过

    # 自定义指令

    # 基本语法

    1. 命名规则：指令命名时使用 -连接，定义时不需要添加 v-，使用时需添加 v-
    2. 局部定义：directives:{ focus:{ bind(el,b){}, inserted(el,b){}, update(el,b){} } } 或简写 directives:{ focus(){ 此时是执行 bind 和 update } }
    3. 全局定义：Vue.directive('focus'，{})
    4. 钩子函数：bind(el,bind)、inserted(el,bind)、update(el,bind)
      1. el：指令所绑定的元素，可以用来直接操作 DOM
      2. binding：包含指令相关属性，常用的是 value ，即获取指令绑定的值

    # 执行流程

    1. bind：只调用一次，指令第一次绑定到元素时调用
    2. inserted：被绑定元素插入父节点时调用
    3. update：指令所在的组件被重新解析时会被调用
