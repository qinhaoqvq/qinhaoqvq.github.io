    Vue组件

    qinhaoqvq Vue

    # el 属性

    提供一个在页面上已存在的 DOM 元素作为 Vue 实例的挂载目标。如果render函数和template property都不存在，挂载 DOM 元素的 HTML 会被提取出来用作模板，此时，必须使用 Runtime + Compiler 构建的 Vue 库

    # template 属性

    一个字符串模板作为 Vue 实例或者组件实例的标识使用。模板将会替换挂载的元素

    # 👺VueComponent

    # ref 属性

    定义在普通DOM元素或自定义组件的属性，Vue 会将使用该属性的元素或组件的引用保存在$.refs中，属性名以ref的值为准。可以通过this.$refs.xxx 获取到该DOM元素或组件实例对象

    1. 返回类型：DOM 元素，即 xxx instanceof HTMLElement
    2. 返回类型：VueComponent 实例，即 xxx instanceof VueComponent

    该属性类似 document.getElementById(id) 的替代

    # props 配置

    props 用于接收来自父组件的数据。props 可以是简单的数组，或者使用对象作为替代，对象允许配置高级选项，如类型检测、自定义验证和设置默认值

    • type
    • default：any
    • required：Boolean
    • validator：Function

    父组件可以给子组件传递『函数类型』的 props，可以实现子组件给父组件传递值

    # mixins 混入

    可以将外界定义的混入对象的组件配置项与引用对象的组件配置项进行『合适的』合并，从而达到 Vue 组件的复用。如果在合并过程中存在同名的选项，那么会以引用对象的配置选项为准。

    1. 局部引入一个混入对象：mixins:[ obj ]
    2. 全局引入一个混入对象：Vue.mixin( obj )，不推荐使用全局引入的方式
    3. 混入对象可以配置data、methods、computed和钩子函数等
    4. 如果混入对象和引用对象都定义了 created 钩子函数，那么两个函数会被依次调用

    # 自定义插件

    Vue 定义插件，要求对象必须要有install() 方法，并且该方法的第一个参数就是 Vue 构造函数对象，之后可以自定义多个参数。然后在创建 Vue 实例对象之前，通过Vue.use(plugin) 调用插件。在自定义插件中，可以做很多事情，比如：

    1. 自定义指令：Vue.directive('my-directive',{bind(el,binding), inserted(...), update(...)})
    2. 定义全局属性或方法：Vue.xxx=/function(){}
    3. 引入一个混入对象：Vue.mixin( obj )
    4. 定义全局过滤器：Vue.filter('my-filter',function(value){return})
    5. 给 Vue 原型对象添加属性或方法：Vue.prototype.xxx=name/function(){}

    # LocalStorage 和 SessionLocalStorage

    1. LocalStorage 可以在浏览器存储（setItem）、读取（getItem）、删除（removeItem）和删除所有（clear），键值对，但是，键和值只能为字符串类型。数据可以长期保存
    2. SessionStorage 用法与 LocalStorage 一致，但是数据是会话级别的，当会话已结束，数据就会被销毁

    # 自定义事件

    自定义事件主要应用于子组件向父组件传递数据的场景。在父组件中为子组件绑定自定义事件，在子组件中触发自定义事件。

    # 基本语法

    1. 父组件使用 v-on: xxx, 监听子组件事件
    2. 子组件使用 $emit('xxx') 触发一个xxx事件，该事件能够被父组件监测到
    3. 子组件使用 $emit('xxx',value) 触发一个xxx事件的同时并抛出一个 value
    4. 父组件使用 $event 能够访问到子组件抛出的 value 值

    # 流程

    # 绑定

    1. v-on 或 @ 进行事件绑定

    2. ref 获取到组件实例对象，然后 this.$refs.vc.$on('my-event',callbacl)，此时回调函数的 this指向触发事件的组件实例对象，如果想要使用父组件的this，可以将回调函数改为** 箭头函数**

    # 触发

    子组件触发事件this.$emit('my-event'，data)

    # 解绑

    this.\$off() 解绑所有事件或this.\$off('my-event')解绑单个事件。解绑事件一般是在钩子函数 beforeDestroy 中进行

    # 销毁

    组件实例销毁后，自定义事件不再生效

    # 组件使用原生DOM事件

    v-bind:event.native="" 或 @event.bative=""

    # 全局事件总线

    可以实现任意组件之间进行通信。 在创建 Vue 实例对象时，通过钩子函数 beforeCreate()中创建全局事件总线，Vue.prototype.$bus = this

    1. 创建：在创建 Vue 实例时，通过钩子函数 beforeCreate 创建 $bus 属性，Vue.prototype.\$bus = this（此时this指向 vm 对象）
    2. 绑定事件：在一个组件中，通过钩子函数 mounted 为 $bus 绑定自定义事件，this.\$bus.\$on('eventName', callback) ，回调函数定义在该组件中，所以该组件能够接收到触发该事件的组件传递的数据
    3. 触发事件：在另一个组件中，通过调用 this.\$bus.\$emit('eventName',data)，触发自定义事件，从而将数据传递给其他组件
    4. 解绑事件：因为 $bus 是一个全局共享的对象，所以在组件即将被销毁前应该解绑在此组件中定义的时间，即，通过钩子函数 beforeDestroy 调用this.\$bus.\$off('eventName')

    # 消息订阅与发布

    可以实现任意组件之间进行通信。需要依赖一些消息与订阅发布的_第三方JS库_，pubsub-js。订阅是数据接收方，发布是数据发送方

    1. 引入模块： import PubSub from 'pubsub-js'
    2. 消息订阅：var token = PubSub.subscribe('MY-TOPIC', callback(topicName, data))
    3. 消息发布
      1. 异步发布： PubSub.publish('MY TOPIC', 'hello world!')
      2. 同步发布： PubSub.publishSync('MY TOPIC', 'hello world!')
    4. 取消订阅：PubSub.unsubscribe(token)取消指定主题，PubSub.clearAllSubscriptions()取消所有订阅的主题

    # 🔔nextTick

    将回调延迟到下次 DOM 更新循环之后执行。在修改数据之后立即使用它，然后等待 DOM 更新。它跟全局方法 Vue.nextTick 一样，不同的是回调的 this 自动绑定到调用它的实例上

    1. 局部延迟回调：vm.$nextTick（callback）
    2. 全局延迟回调：Vue.nextTick（callback，context）

    # 过渡与动画

    参考资料 (opens new window)

    Vue 在插入、更新或者移除 DOM 时，提供多种不同方式的应用过渡效果

    • 在 CSS 过渡和动画中使用自定义 class
    • 可以配合使用第三方 CSS 动画库，如 Animate.css
    • 在过渡钩子函数中使用 JavaScript 直接操作 DOM
    • 可以配合使用第三方 JavaScript 动画库，如 Velocity.js

    # 过渡

    # 介绍

    通过<transition>或<transition-group>标签可以实现给任何 DOM 元素或组件添加过渡效果。过渡效果在插入或删除在 transition 中的元素或组件时触发，处理流程如下：

    1. 自动嗅探目标元素是否应用了 CSS 过渡或动画，如果是，在恰当的时机添加/删除 CSS 类名。
    2. 如果过渡组件提供了 JavaScript 钩子函数，这些钩子函数将在恰当的时机被调用。
    3. 如果没有找到 JavaScript 钩子并且也没有检测到 CSS 过渡/动画，DOM 操作 (插入/删除) 在下一帧中立即执行。

    # 过渡的默认类名规则

    1. v-enter：定义进入过渡的开始状态
    2. v-enter-active：定义进入过渡生效时的状态
    3. v-enter-to：定义进入过渡的结束状态
    4. v-leave：定义离开过渡的开始状态
    5. v-leave-active：定义离开过渡生效时的状态
    6. v-leave-to：定义离开过渡的结束状态

    对于这些在过渡中切换的类名来说，如果你使用一个没有名字的 <transition>，则 v- 是这些类名的默认前缀。如果你使用了 <transition name="my-transition">，那么 v-enter 会替换为 my-transition-enter

    # 过渡的自定义类名

    可以通过以下 attribute 来自定义过渡类名，它们的优先级高于普通的类名

    • enter-class
    • enter-active-class
    • enter-to-class
    • leave-class
    • leave-active-class
    • leave-to-class

    # 动画

    CSS 动画用法同 CSS 过渡，区别是在动画中 v-enter 类名在节点插入 DOM 后不会立即删除，而是在 animationend 事件触发时删除。可以只是用 v-enter-active 和 ** v-leave-active**

    # 初始渲染的过渡

    可以通过 appear attribute 设置节点在初始渲染的过渡

    <transition appear>
  <!-- ... -->
</transition>

    # 插槽 slot

    slot 是 Vue 实现了一套内容分发的 API，将 <slot> 元素作为承载分发内容的出口。简单来说，如果想通过在组件标签内添加内容到组件中指定位置，那么就需要使用到<slot> 标签。首先在组件中的template中定义好<slot>标签，之后，Vue 在解析时会将组件标签内定义的内容进行解析转换，然后将解析生成的DOM元素替换组件中的<slot>标签

    在 Vue 2.6.0 中，为具名插槽和作用域插槽引入了一个新的统一的语法 (即 v-slot 指令)。它取代了 slot 和 slot-scope 这两个目前已被废弃但未被移除且仍在文档中 (opens new window)的 attribute，但已经被官方废弃且不会出现在 Vue 3 中

    # 默认插槽

    1. 在组件内定义 slot 时，可以不用指定 name 属性，其默认值为 default
    2. 在组件中插入 slot 是，可以不用指定 slot 属性，其默认值为 default

    # 具名操作

    # 旧API

    1. 在组件内定义 slot 时，使用 name 属性设置插槽名称
    2. 在组件中插入 slot 时，使用 slot = "slot-name" 属性指定要将组件标签内的内容插入到的插槽

    # 新API

    1. 在组件内定义 slot 时，使用name 属性设置插槽名称
    2. 在组件中插入 slot 时，需要使用 template 标签包裹插入内容，然后通过 v-slot:slotname属性指定插槽。默认插槽可以写为 v-slot:default 或 v-slot

    # 作用域插槽

    让插槽内容能够访问子组件中才有的数据，绑定在 <slot> 元素上的 attribute 被称为插槽 prop。在父级作用域中，我们可以使用带值的 v-slot或slot-scope 来定义我们提供的插槽 prop

    # 旧API

    1. 在组件内定义 slot 时，使用 name 属性设置插槽名称，使用v-bind:props="data" => 插槽 prop
    2. 在组件中插入 slot 时，使用slot = "slot-name" 属性指定插槽，然后通过 slot-scope="{props}"，属性接收插槽prop

    # 新API

    1. 在组件内定义 slot 时，使用 name 属性设置插槽名称，使用v-bind:props="data" => 插槽 prop
    2. 在组件中插入 slot 时，需要使用 template 标签包裹插入内容，然后通过 v-slot:slotname="{props}"，指定使用的插槽，并且定义props对象，用来接收插槽prop的数据

    # 代码验证

    <!-- 在组件中定义插槽 -->
<template>
  <div>
    <h3>NameSlot</h3>
    <header>
      <slot name="header" :person="person">NameSlot header</slot>
    </header>
    <main>
      <!-- 默认插槽 -->
      <slot>NameSlot main</slot>
    </main>
    <footer>
      <slot name="footer">NameSlot footer</slot>
    </footer>
  </div>
</template>

<!-- 在父组件中使用组件并传递内容-->
<div>
<NameSlot>
  <!-- v-slot/v-slot:default -->
  <template v-slot>
    <p>父组件的 NameSlot</p>
  </template>
</NameSlot>
<NameSlot>
  <template v-slot:header>
    <p>父组件的 NameSlot</p>
  </template>
</NameSlot>
<NameSlot>
  <template v-slot:header="{ person }">
    <p>父组件的 NameSlot, slot的数据 {{ person.name }}</p>
  </template>
</NameSlot>
</div>
