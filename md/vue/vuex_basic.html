    Vuex 状态管理库

    vuePress-theme-reco qinhaoqvq    2022

    Vuex 状态管理库

    qinhaoqvq Vue

    # 版本问题

    在 Vue2 中使用 Vuex 3，在 Vue3 中使用 Vuex4

    # 是什么

    Vuex 是 Vue 官方提供的一个状态管理模式 + 库的插件。把组件的共享状态抽取出来，以一个全局单例模式管理。在这种模式下，我们的组件树构成了一个巨大的“视图”，不管在树的哪个位置，任何组件都能获取状态或者触发行为！这就是" 状态管理模式"

    # 能干嘛

    1. 多层嵌套的组件、兄弟组件间的状态会更好管理维护。
    2. 缓存一些当前要使用请求远程或本地的数据集（刷新后会自己销毁）。
    3. 有了第二条，就可以减少向服务器的请求，节省资源。
    4. 对开发者来说，如果你的项目足够复杂，团队的规模也不仅是一个人，数据集中处理更利于程序的稳定和维护

    # 👺核心概念

    # Store

    每一个 Vuex 应用的核心就是 store（仓库），"store" 基本上就是一个容器，它包含了 state、getters、mutations、actions、modules

    1. Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。
    2. 你不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是显式地提交 (commit) mutation。这样使得我们可以方便地跟踪每一个状态的变化，从而让我们能够实现一些工具帮助我们更好地了解我们的应用。

    # State

    1. Vuex 就是一个仓库，仓库里面放了很多对象。其中 state 就是数据源存放地。相当于 vm 里面的 data

    2. state 里面存放的数据是响应式的，Vue 组件从 store 中读取数据，若是 store 中的数据发生改变，依赖这个数据的组件也会重新解析渲染

    # Getter

    1. getters 可以对 state 进行计算操作。相当于 vm 里面的 computed 属性
    2. 虽然在组件内也可以做计算属性，但是 getters 可以在多组件之间复用
    3. 如果一个状态只在一个组件内使用，是可以不用 getters

    # Mutation

    1. 它是修改 state 中状态的唯一入口。想要修改 state 中的状态，必须提交一个 mutation，无论是在 action 中提交，还是在组件中直接调用$sotre.commit()进行提交
    2. mutation 必须是同步函数
    3. mutation 函数，包括两个参数，分别是 state 对象和通过 commit传入的 value

    # Action

    1. 可以通过 dispatch 分发 action
    2. 可以通过 commit 提交mutation，而不是直接维护 state 对象
    3. action 可以包含异步操作
    4. action 函数，包括两个参数，分别是 context 上下文对象和通过 dispatch传入的 value

    # 🤡Module

    参考资料 (opens new window)

    Vuex 允许我们将 store 分割成模块（module）。每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块

    # 全局命名空间的模块

    1. 在模块内部，mutation 和 getter 接收到的第一个参数是模板的局部状态对象（即该模块中定义的 state）
    2. 在模块内部，action 接收到的第一个参数是context对象，它可以通过 context.state访问局部状态对象，也可以通过context.rootState访问根节点的状态对象
    3. 如果希望使用全局 state 和 getter，rootState 和 rootGetters 会作为第三和第四参数传入
    const moduleA = {
    state: () => ({}),
    mutations: {},
    actions: {},
    getters: {}
}

const moduleB = {
    state: () => ({}),
    mutations: {},
    actions: {}
}

const store = createStore({
    modules: {
        NameA: moduleA,
        NameB: moduleB
    }
})

store.state.NameA // -> moduleA 的状态
store.state.NameB // -> moduleB 的状态

    # 独立命名空间的模块

    通过namespaced:true属性，使其变为拥有独立命名空间的模块，当模块被注册后，它的所有 getter、action 及 mutation 都会自动根据模块注册的路径调整命名，使用时需要添加空间名前缀

    1. 在独立命名空间模块访问全局内容
      1. （action），接收到的第一个 context 参数中包含 rootState 和 rootGetters
      2. （getter），rootState 和 rootGetters 会作为第三和第四参数传入
      3. （mutation），不能获取全局的内容。理解：获取到全局状态也无法进行修改
    2. 需要在全局命名空间内分发 action 或提交 mutation，将 { root: true } 作为第三参数传给 dispatch 或 commit
    3. 由 2 可知，在模块中可以通过 action 进行转发（dispatch）和提交（commit mutation）
    // person 模块
const state = {
    userList: []
}
const mutations = {
    SAVEUSER(state, value) {
        state.userList.push(value)
    }
}
const actions = {
    saveUser({commit}, value) {
        commit('SAVEUSER', {id: Date.now(), name: value})
    },
    globalDispatchIncrement({dispatch}, value) {
        dispatch('counter/increment', value, {root: true})
    },
    globalCommitIncrement({commit}, value) {
        commit('counter/INCREMENT', value, {root: true})
    }
}

// counter 模块
const mutations = {
    INCREMENT(state, value) {
        state.num += value
    },
    DECREMENT(state, value) {
        state.num -= +value
    }
}
const actions = {
    increment({commit}, value) {
        commit('INCREMENT', value)
    },
    decrement({commit}, value) {
        commit('DECREMENT', value)
    }
}

    ├── index.html

├── api
│   └── ... # 抽取出API请求
├── components
│   ├── App.vue
│   └── ...
└── store
    ├── index.js          # 我们组装模块并导出 store 的地方
    ├── actions.js        # 根级别的 action
    ├── mutations.js      # 根级别的 mutation
    └── modules
        ├── cart.js       # 购物车模块
        └── products.js   # 产品模块

    # 组件绑定的辅助函数

    第一个参数是可选的，指定命名空间的字符串

    第二个参数是对象或字符串数组

    其中 mapActions 和 mapMutations 是比较特殊的，它们生成的函数时有参的，

    name:function(value){this.$store.dispatch('functionName',value)}

    name:function(value){this.$store.commit('functionName',value)}

    所以，在调用它们生成的函数时，需要通过括号直接传递参数。当然，也可以通过迂回的方式再定义一个无参的函数包括它，再进行调用
