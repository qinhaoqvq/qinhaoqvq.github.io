    # 基础

    # 数据类型

    # 基本类型

    1. String
    2. Number
    3. boolean
    4. undefined
    5. null
    # Number
    1. Infinity表示一个正无穷，-Infinity表示一个负无穷。它是一个字面量，typeof 返回 number
    2. NaN，它是一个特殊的数字，表示 Not A Number。它是一个字面量，typeof 返回 number

    # 对象类型

    1. Object：任意对象
    2. Function：一种特别的对象，内部保存的可执行代码，可调用执行
    3. Array：对象的集合

    # 类型判断

    1. typeof
    2. instanceof
    3. ===

    # undefined和null的区别

    undefined：定义了变量但未赋值

    null：定义了变量并赋值了，赋值为null

    # 字面量

    理解为用来为变量赋值时的常数量

    1. 数字字面量，var obj = 1
    2. 字符串字面量，var obj = 'str'
    3. 对象字面量，var obj = {}

    # 伪数组

    伪数组，也称为类数组。长相很像数组，但是将他的原型_proto_类型是Object，数组的是Array，它没有数组的splice，concat，pop等方法

    # 作用

    伪数组存在的意义，是可以让普通的对象也能正常使用数组的很多方法

    # 特点

    1. 具有length属性
    2. 按索引方式存储数组
    3. 不具有数组的方法，比如push()，pop()等

    # 常见的伪数组

    1. jQuery 对象（比如 $(“div”) ）
    2. 函数内部的 arguments
    3. DOM 对象列表（document.childNodes返回的NodeList对象）
    4. 上传文件时选择的file对象也是伪数组

    # 伪数组转为真正数组

    1. Array.prototype.slice.call(array)或[].slice.call()，两者都一样，原理: 原型链
    2. 使用ES6的Array.from()
    3. 使用ES6的扩展操作符let real = [...array]

    # 总结

    1. 伪数组没有数组 Array.prototype 的属性值，类型是 Object ，而数组类型是 Array
    2. 数组是基于索引的实现， length 会自动更新，而对象是键值对
    3. 使用对象可以创建伪数组，伪数组可以利用call或者apply很方便的转化为真数组

    # 变量提升

    # arguments

    在函数调用时，解析器每次都会向函数传递两个隐含的参数（this 和 arguments）。arguments 是封装实参的对象，是一个类数组对象（类似数组的对象，但不是数组，Array.isArray(argumnets) == false ），即它可以通过索引来操作数据。通过arguments.callee方法可以获取到当前执行函数的对象

    4. function(){}是一个匿名函数

    # 函数调用

    JavaScript 函数有 4 种调用方式，每种方式的不同在于 this 的初始化。

    1. 作为一个函数调用『函数作为全局对象调用，会使 this 的值成为全局对象，即 window』

      function func(param){}
// 调用函数
func('abc')
// 上函数不属于任何对象。但是在 JavaScript 中它始终是默认的全局对象, 即, window.func('abc')。
// 此时 this 指向 window 对象

    2. 函数作为方法调用『函数作为对象方法调用，会使得 this 的值成为对象本身』

      var obj = {
  name : 'admin',
  say : function(){ console.log('hello world') }
}
// 调用函数
obj.say();

    3. 使用构造函数调用函数『函数作为构造方法调用，this 的值为通过调用函数实例化出的对象（new object）』

      function Person(name,age){this.name=name,this.age=age}
// 调用函数
var p = new Person('admin'，18)

    4. 作为函数方法调用函数『 在调用函数时第一个参数会成为 this 的值， 即使该参数不是一个对象』

      // call() 和 apply() 是预定义的函数方法。 两个方法可用于调用函数，两个方法的第一个参数必须是对象本身。
// 1. call()
function myFunction(a, b) {
    return a * b;
}
myObject = myFunction.call(myObject, 10, 2);     // 返回 20
// 2. apply()
function myFunction(a, b) {
    return a * b;
}
myArray = [10, 2];
myObject = myFunction.apply(myObject, myArray);  // 返回 20


var myObject = {} // ** myObject 没有 MyFunction 方法，但是可以通过 call/apply 方式调用 **
myFunction.call(myObject, 1, 2)

// 两个方法都使用了对象本身作为第一个参数。 两者的区别在于第二个参数： apply传入的是一个参数数组，也就是将多个参数组合成为一个数组传入，而call则作为call的参数传入（从第二个参数开始）

    # 🍿this

    解析器在每次调用函数时都会向函数内部传递一个隐式参数，即this，this 指向一个对象，这个对象我们称为函数执行的上下文对象。根据调用方式不同，this会指向不同的对象。

    # 🥛原型与原型链

    1. 所有的 JavaScript 对象都会从一个 prototype（原型对象）中继承属性和方法（prototype）
    2. JavaScript 对象有一个指向一个原型对象的链。当试图访问一个对象的属性时，它不仅仅在该对象上搜寻，还会搜寻该对象的原型，以及该对象的原型的原型，依次层层向上搜索，直到找到一个名字匹配的属性或到达原型链的末尾。一般情况，Object 为最顶端原型对象

    1. 每创建一个函数，解析器都会向函数中添加一个prototype属性。每一个函数都有一个属于自己的原型对象
    2. 通过构造函数创建对象时，该对象中都会有一个隐式的__proto__属性，该属性与创建对象的函数对象的prototype属性相同
    3. 使用 in 检查对象中是否含有某个属性时，如果对象中没有该属性但是原型对象中有，则返回 true
    4. 使用 hasOwnProperty（）方法可以检查对象本身是否含有某个属性

    # 作用

    创建构造函数时，可以将公共的属性和方法定义到原型对象中。这样可以不需要在每个对象中重复创建，并且也不会影响到全局作用域

    # 显示原型

    每个函数 function 都有一个 prototype 属性，即，显示原型。

    在定义函数式自动添加，默认值为空 Object 对象

    # 隐式原型

    每个实例对象都有一个 __proto__ 属性，即，隐式原型。

    在创建对象时自动添加，默认值为构造函数的 prototype 属性

    # 🌈闭包

    # 定义

    闭包是有权访问另外一个函数中的局部变量的函数。

    通常可以理解为，在一个函数中能够访问另外一个函数中的局部变量

    # 原理

    闭包的实现原理，其实是利用了作用域链的特性，我们都知道作用域链就是在当前执行环境下访问某个变量时，如果不存在就一直向外层寻找，最终寻找到最外层也就是全局作用域，这样就形成了一个链条。

    # 作用

    **作用1：**隐藏变量，避免全局污染

    **作用2：**可以读取函数内部的变量，延长了变量的作用范围

    **缺点1：**导致变量不会被垃圾回收机制回收，造成内存消耗

    **缺点2：**不恰当的使用闭包可能会造成内存泄漏的问题

    # 常见场景

    1. for 循环创建异步任务并且使用到遍历变量 i 时
    2. 闭包返回一个 function，并且在该函数中使用了 this，此时 this 指向的相当于是创建该 function 的上下文对象
