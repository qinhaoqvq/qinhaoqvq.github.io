    # JavaScript 运行环境

    # 浏览器

    • 浏览器是 JavaScript 的前端运行环境
    • 浏览器可以运行 JavaScript 代码，主要是浏览器提供了运行环境。其中，最核心的是 JavaScript 执行引擎和浏览器内置 API

    # Node.js

    • Node.js 是 JavaScript 的后端运行环境
    • Node.js 也可以运行 JavaScript 代码，它是一个基于 Chrom V8 引擎的 JavaScript 运行环境。其中，最核心的是 JavaScript 执行引擎和Node.js 内置 API
    • Node.js 无法调用浏览器内置 API，例如 DOM、BOM

    # Node.js 简介

    • Node.js 是一个能够在服务器端运行 JavaScript 的开源的、跨平台的 JavaScript 运行环境
    • Node.js 采用 Google 的 V8 引擎运行 js 代码，使用事件驱动、非阻塞和异步I/O模型等技术来提高性能

    # 内置 API 模块

    # fs 文件系统模块

    # path 路径模块

    # http 模块

    # Node.js 模块化

    # 模块分类

    1. 内置模块（fs、path、http）
    2. 自定义模块（用户创建的每个 js 文件）
    3. 第三方模块（由第三方开发出来的模块，使用前需要先下载）

    # 加载模块

    1. 使用 require() 方法加载其他模块时，会执行被加载模块中的代码
    2. 使用 require() 方法加载自定义模块时，可以省略 .js 文件名后缀

    # 模块作用域

    1. 防止全局变量污染的问题

    # 向外共享模块作用域中的成员

    # module 对象

    每个 .js 自定义模块中都有一个 module 对象，它里面存储了和当前模块有关的信息。其中就有一个 reports 属性，其默认值为 {}

    # module.exports 对象

    在自定义模块中，可以使用 module.exports 对象，将模块内的成员共享出去，供其他模块使用。即，当使用 require() 方法导入自定义模块时，就是得到了 module.exports 指向的对象

    # exports 对象

    Node.js 提供了 exports 对象，默认情况下 exports 和 module.exports 指向同一个对象。最终结果还是 module.exports 指向的对象

    # CommonJS 规定

    1. 每个模块内部，module 变量代表当前模块
    2. module 变量是一个对象，它的 exports 属性，即 module.exports 是对外的接口
    3. 使用 require() 方法加载某个模块，其实就是加载该模块的 module.exports 属性

    # npm 与包

    # 包

    1. 第三方模块也被称为包。都是第三方开源的、免费的
    2. 包是基于 Node.js 内置模块封装出来的，提供更高级的、更方便的 API。包与内置模块的关系，类比 jQuery 与浏览器内置API
    3. https://www.npmjs.com/

    # npm

    • 安装包命令: npm install 包完整名称，简写 npm i 包完整名称
    • 安装指定版本的包使用@指定版本号，npm i jQuery@1.10.1
    • node_modules: 该文件夹用来存放所有已安装到项目中的包。当 require() 引入第三方包时，就是从该目录中查找并加载的
    • package-lock.json: 该配置文件用来记录 node_modules 目录下的每一个包的下载信息，例如包名、版本号、下载地址等信息
    # 包的语义化版本规范

    包的版本号以'点分十进制'形式进行定义，总共有三位数字

    1. 第1位数字: 大版本
    2. 第2位数字: 功能版本
    3. 第3位数字: Bug修复版本

    # 包管理配置文件

    在项目根目录中，必须有一个叫做 package.json 的包管理配置文件

    1. 项目的名称、版本号、描述等
    2. 项目中都用到了哪些包
    3. 哪些包只在开发期间会用到
    4. 哪些包在开发和部署是都需要用到

    # dependencies

    记录项目中已经安装的包

    # 一次性安装所有的包

    npm install，执行该命令时，npm 包管理工具会先读取 package.json 中的 dependencies 节点，获取项目中所有依赖包的名称和版本信息，然后 npm 包管理工具会把这些包一次性下载到项目中

    # 卸载包

    npm uninstall 包完整名称，包被卸载完成后，node_modules 目录下会删除对应包文件，package.json 也会删除对应依赖

    # devDependencies 节点

    如果某些包只在项目开发阶段使用，则建议把这些包记录到 devDependencies 节点中

    （简写）npm install 包完整名称 -D
（全写）npm install 包完整名称 --save-dev

    # npm 镜像服务器

    下载慢的原因：海底光缆

    # 切换 npm 下包镜像源

    npm config get registry // 查看
npm config set registry=https://registry.npm.taobao.org/ //设置

    # nrm

    1. 全局安装 ：npm -i nrm -g
    2. 查看所有可用的镜像源：nrm ls
    3. 切换镜像源：nrm use taobao

    # 包的分类

    1. 项目包（存放在 node_modules）
      1. 开发依赖包（devDependencies）
      2. 核心依赖包（dependencies）
    2. 全局包（${USER}/**/node_modules）

    # 规范的包结构

    1. 包必须以独立的目录存在
    2. 在包的根目录下必须包含 package.json
    3. package.json 必须包含 name 、version、main，分别是包名字、版本号、包的入口

    # 发布包

    1. 注册 npm 账号
    2. 登录 npm 账号 npm login。需要使用官方镜像服务器
    3. 发布 npm 包，在包的根目录，执行npm publish。发布的包名不能与npm包服务器已有的包名字重复
    4. 删除已发布的包，执行 npm unpublish 报名 --force
      1. npm unpublish 只能删除72小时内发布的包
      2. 24小时内不允许重复发布已删除的包

    # 模块的加载机制

    模块在第一次加载后会被缓存（模块第一次被加载会执行该模块的代码）

    1. 内置模块的加载优先级是最高的
    2. 自定义模块加载时，必须指定以'./'或'../'开头的路径标识符，否则会把它当做内置模块或者第三方模块
      1. 第一步，按照完整的文件名进行加载
      2. 第二步，补全 .js 扩展名进行加载
      3. 第三步，补全 .json 扩展名进行加载
      4. 第四步，补全 .node 扩展名进行加载
      5. 如果还未加载成功，则加载失败，报错
    3. 第三方模块加载时，如果在当前项目的 node_modules 没有找到对应的第三方模块时，会依次向父级目录中的 node_modules 下查找并加载，一直到根目录，如果还未加载到则报错
    4. 目录作为模块
      1. 读取 package.json 的 main 属性
      2. 加载 index.js
      3. 报错，can not found module
