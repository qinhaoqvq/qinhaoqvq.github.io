    第14章 类型信息

    qinhaoqvq

    # 概念

    # 静态类型语言和动态类型语言

    • 静态类型语言：在编译期进行类型检查的语言，如Java、C++

      • 变量的数据类型在编译期声明以后，不能再改变其数据类型，即变量在使用前必须先声明类型。类型检查在编译期进行，好处是编译期提前检查可能出现的错误

    • 动态类型语言：在运行期进行类型检查的语言，如JavaScript、Python

      • 在运行期检查变量类型的语言，也就是在编写代码的时候可以不指定变量的数据类型，例如var num=1。动态类型语言中，变量没有类型，而值有类型

    # 类型检查和类型转换的区别

    • 类型检查：发生在编译期，检查方法是不是接收到了合适类型的参数，赋值是不是有合适的右值
    • 类型转换正确性检查：发生在运行期，检查一个类型转换到另外一个类型是不是正确合法，如果执行了一个错误的类型转换，就会抛出一个ClassCastException异常

    # Java语言性质

    • Java是静态类型语言，因为其在编译期接受静态的类型检查
    • Java是动态类型安全的语言，因为其运行期进行类型转换正确性检查

    # RTTI和反射

    ✨运行时类型信息使得你可以在程序运行时发现和使用类型信息。在Java中运行时识别对象和类的信息的方式有两种：传统的RTTI和反射

    # 传统RTTI

    # 是什么

    RTTI（Runtime Type Identification）运行阶段类型识别，含义就是在运行时，识别一个对象的类型。他使得从只能从编译期执行面向对象类型的操作的禁锢中解脱出来，并且可以使用某些非常强大的程序

    # Java执行RTTI方式

    1. 传统的类型转换，由RTTI确保类型转换的正确性，如果执行了错误的类型转换，会抛出一个ClassCastException的异常。
    2. 代表对象类型的Class对象，通过Class对象获取所需要的信息
    3. 类型转换前用instanceof检查

    # Class对象

    # 是什么

    每个类经过编译都会生成一个.class文件，这个.class文件就是表示类信息的二进制字节码文件。运行时，虚拟机的类加载器会加载这个.class文件，生成Class对象，并将.class文件中的常量池放入方法区的运行时常量池，Class对象就代表着类型信息，并提供访问运行时常量池的途径

    # 类字面常量

    例如：Class clazz = Object.class

    # ✨Class.forName()和类字面常量的区别

    1. 使用.class来创建Class对象的引用时，不会自动地初始化该Class对象，并且在编译器就已经做了类型检查
    2. Class.forName()创建Class对象引用时，会自动地初始化该Class对象

    # 代码验证

    public class StaticFinalValue {

    // static和final 的编译时常量
    static class StaticFinalConstant {
        static final int num = 0;
        static {
            System.out.println("StaticFinalConstant Construct");
        }
    }

    // static和final 的非编译时常量
    static class StaticFinalVariable {
        static final Date num = new Date();
        static {
            System.out.println("StaticFinalVariable Construct");
        }
    }

    // static 的常量
    static class StaticVariable {
        static int num = 1;
        static {
            System.out.println("StaticVariable Construct");
        }
    }

    public static void main(String[] args) {
        System.out.println(StaticFinalConstant.num);
        System.out.println(StaticFinalVariable.num);
        System.out.println(StaticVariable.num);
    }
}
// 输出结果: 
0
StaticFinalVariable Construct
Sun Mar 27 13:03:09 CST 2022
StaticVariable Construct
1

    # 总结

    1. 如果一个static final的值是编译期常量，那么访问这个值就不需要对其所在的Class对象进行初始化。如static final int num = 0;

    2. 如果一个static final的值是非编译期常量，那么访问这个值将强制进行Class对象的初始化。如

      static final Date now = new Date();

    3. 如果一个static非final的值，那么访问这个值时，需要先进行Class对象初始化。如static int num = 1

    # 泛型与Class对象

    向Class引用添加泛型语法是为了提供编译期的类型检查，因此如果你操作有误，稍后立即就会发现这一点

    1. 通配符?：表示任何事物。使用Class<?>优于Class
    2. 通配符extends：限定Class应用为某种类型或者该类型的任何子类型。Class<? extends Number> class

    # 反射

    # 是什么

    ✨它是一种允许程序在运行时通过Reflection APIs访问任何一个已知名称的class的内部信息（包括其modifiers、 superclass、实现的 interfaces 、也包括其fields和methods的所有信息，并可于运行时改变fields内容或调用methods）的机制。

    # 动态代理

    Java动态代理是在代理模式的基础上的升级，它可以动态的创建代理并动态的处理对所代理方法的调用。动态代理可以将所有调用（也可以过滤某些方法）重定向到代理类，因此代理类的构造器通常会传递一个"实际"对象的应用。

    • Proxy.newProxyInstance()
    • InvocationHandler接口

    # 使用反射越过编译器检查

    public class ReflectionOverCheck {

    public static void main(String[] args) throws Exception {
        List<String> stringList = new ArrayList<>();
        stringList.add("HelloWorld");
        Class<? extends List> clazz = stringList.getClass();
        Method addMethod = clazz.getMethod("add", Object.class);
        // 通过定义容器和添加Java泛型, 是的虚拟机在编译器间可以做类型检查,但是通过反射机制可以绕开该检查。那是因为反射运行时才会加载.class文件
        addMethod.invoke(stringList, 100);
        System.out.println(stringList);
    }
}
// 输出结果:
[HelloWorld, 100]

    # RTTI和反射的本质区别

    • RTTI：编译器在编译时打开和检查.class文件，也就是可以用'普通的RTTI'方式调用对象的所有方法
    • 反射：对于反射机制来说，.class文件在编译时是不可获取的，只有在运行时才会打开和检查.class文件
