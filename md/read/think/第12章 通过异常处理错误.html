    第12章 通过异常处理错误

    qinhaoqvq

    # 概念

    # 异常理解

    异常与语法错误是不同的概念，语法错误代码将会报红，编译不通过，而异常是可能会发生但不一定会发生的错误

    # 异常情形

    它是指阻止当前方法或作用域继续执行的问题

    # 异常抛出流程

    1. 使用new在堆上创建异常对象（即，分配存储空间和调用构造方法）
    2. 当前执行代码被终止
    3. 从当前环境中弹出异常对象的引用
    4. 异常处理程序处理该异常

    # Java标准异常

    # 分类

    # 基于继承关系

    • Throwable：Java中所有异常类的父类。主要分为Error和Exception
    • Error：用来表示编译时和系统错误，与程序代码无关
    • Exception：可以被抛出的基本类型。主要包含RuntimeException和非RuntimeException
    • RuntimeException：一般由程序逻辑错误引起，程序应该从逻辑角度尽可能避免这类异常的发生
    • 非RuntimeException：一般是指必须进行处理的异常，如果不处理，程序就不能编译通过

    # 基于编译器是否检查

    • 被检查的异常（非RuntimeException类）

      checked异常在编译时检查，这意味着如果一个方法抛出checked异常，那么它应该使用try-catch块或者使用throws关键字来处理这个异常，否则的话程序会报编译错误

    • 不被检查的异常（RuntimeException类）

      unchecked异常在编译时不会检查，这意味着即使你没有声明或者处理异常你的程序也会抛出一个unchecked异常，程序不会给出一个编译错误，大多数情况下这些异常的发生是由于用户在交互过程中提供的坏数据

    # 异常处理

    # 能干嘛

    异常处理的一个重要目标就是把错误处理的代码和错误发生的地点分离开来，这样一来，主干代码就不会与错误处理代码混在一起，提高代码的可读性、可维护性

    # 捕获异常

    使用try块来捕获异常

    # 异常处理程序

    抛出的异常必须在某处得到处理，这个地点就是catch块（异常处理程序）

    1. 异常处理机制会按照代码顺序找出最近的、能匹配的（查找时并不要求抛出异常对象的类型与异常处理程序声明的异常类型完全匹配，是子类其也可以）
    2. 找到匹配的异常处理程序之后，它就认为系统将得到处理，就不会再继续查找

    # finally

    try关键字最后可以定义finally代码块。finally块中定义的代码，总是在try和任何catch块之后、方法完成之前运行。在方法return之后finally字句也是执行的

    # 使用场景

    因为不管是否发生异常finally都会执行，因此我们可以在finally代码块中执行关闭网络连接、关闭已打开的文件和释放线程的的操作

    # 使用经验

    构造器中使用try-catch-finally语句时，最安全的使用方式是使用嵌套的try语句。或者使用try-with-resources语句

    public class ConstructorUseTry {
    public ConstructorUseTry() {
        try {
            BufferedReader reader = new BufferedReader(new FileReader("filepath"));
            try {
                String line = reader.readLine();
            } catch (IOException e) {
              	// business exception
                e.printStackTrace();
            } finally {
                reader.close();
            }
        } catch (IOException e) {
          	// reader constructor exception
            e.printStackTrace();
        }
    }
}
// 或者
public ConstructorUseTry() {
  try (BufferedReader reader = new BufferedReader(new FileReader("filepath"))) {
            String line = reader.readLine();
        } catch (Exception e) {
            e.printStackTrace();
        }
}
