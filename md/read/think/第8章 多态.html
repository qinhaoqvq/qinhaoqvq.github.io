    # 什么是多态

    面向对象的三大特性：封装、继承、多态。从一定角度来看，封装和继承几乎都是为多态而准备的

    # ✨多态是什么

    • 多态是同一行为具有多个不同表现形式的能力
    • 多态就是同一个接口，使用不同的实例而执行操作不同
    • 指允许不同类的对象对同一消息做出响应。即同一消息可以根据发送对象的不同而采用多种不同的行为方式（发送消息就是函数调用）

    # ✨实现多态的技术

    • 动态绑定（dynamic binding），是指在运行期间判断所引用对象的实际类型，根据其实际的类型调用其相应的方法

    # 多态的作用

    • 消除类型之间的耦合关系。即允许多种类型（子类）视为同一类型（基类）来处理

    # 多态例子

    1. 几何图形（基类），圆形、矩形（子类）
    2. 在不同软件中，F1功能不同

    # ✨多态存在的必要条件

    • 类继承或接口实现
    • 重写
    • 向上转型upcast（父类引用指向子类对象）

    # ✨多态的实现方式

    重载（编译期多态），重写（运行时多态）

    • 同一类中进行方法重载
    • 接口实现
    • 继承父类并进行方法重写

    # ✨多态问题

    # 覆盖私有方法

    // 父类
public class PrivateFather {
    private void say() {
        System.out.println("Father#say()");
    }
}
// 子类
public class PrivateChild extends PrivateFather {
    public void say() {
        System.out.println("Child#say()");
    }
    public static void main(String[] args) {
        PrivateFather father = new PrivateChild();// 父类引用指向子类对象
        // father.say(); Error, 只有非private方法才可以被覆盖, private都隐式的final方法
    }
}

    # 结论

    • 只有非private方法才可以被覆盖，private方法都是隐式final方法，还有就是覆盖private方法的现象编译器不会报错，但是执行结果跟我们预期的却不一样。

    # 域或静态方法

    // 父类
public class StaticFather {
    public int num = 0;
    public static int staticNum = 0;

    public int getNum() {
        return num;
    }
    public static void say() {
        System.out.println("Father#say()");
    }
}
// 子类
public class StaticChild extends StaticFather {
    public int num = 1;
    public static int staticNum = 1;

    @Override
    public int getNum() {
        return num;
    }
    public static void say() {
        System.out.println("Child#say()");
    }
  
    public static void main(String[] args) {
        StaticFather father = new StaticChild();
        System.out.println("filed num: " + father.num + "\n\rmethod num:" + father.getNum());
        // static member not suggest accessed via instance reference, 编译器不建议通过实例访问静态成员
        father.say();
        System.out.println(father.staticNum);
    }
}
// 输出
// filed num: 0
// method num:1
// Father#say()
// 0

    # 结论

    • 当发生向上转型时（Father f = new Child），任何成员变量访问操作都将由编译器解析，因此不是多态的。f.num和c.num分配了不同的存储空间，c实际包含两个num变量，而在引用c.num时实际调用c对象本身的，如果要获取父类f中的num，就必须显示指明super.num。
    • 静态域和静态方法是类本身的，而并非与单个的对象相关联

    # 构造方法中调用方法

    // 父类
public class ConstructFather {
    public ConstructFather(){
        print();
    }
    void  print() {
        System.out.println("Father#print()");
    }
}
// 子类
public class ConstructChild extends ConstructFather {
    private int num = 1;

    public ConstructChild(int num) {
        this.num = num;
        System.out.println("Child#constructor(), num: " + this.num);
    }

    @Override
    void print() {
        System.out.println("Child#print(), num: " + this.num);
    }

    public static void main(String[] args) {
        ConstructFather father = new ConstructChild(5);
    }
}

    # 思考

    # 结论

    • 构造方法中调用可以被子类覆盖的函数时，它会调用子类中的方法，而不是本类的方法
    • 调用子类中的print打印0而不是1，是因为初始化顺序导致的
    1. 在其他所有事物发生之前，将分配给对象的存储空间初始化成二进制的零
    1. 调用父类的构造方法。此时如果在构造方法中调用的方法被子类覆盖，则会调用子类的方法
    2. 按照声明顺序调用成员的初始化方法
    3. 调用子类的构造方法
