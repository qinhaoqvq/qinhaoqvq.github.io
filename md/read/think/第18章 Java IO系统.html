    # "流"概念

    I/O类库中常使用流这个抽象概念，它代表任何有能力产出数据的数据源对象或者有能力接收数据的接收端对象。流屏蔽了实际I/O设备中处理数据的细节

    # File类

    # 目录列表器

    • File.list()：不带参数的list()方法可以获得此File对象包含的全部文件列表
    • File.list(FilenameFilter)：带参数的list()方法可以过滤出符合自定义规则的文件列表

    # I/O类继承结构

    # InputStream和OutputStream

    # 概念

    Java类库中的I/O类分成输入和输出两部分

    • 任何自InputStream 或 Reader派生而来的类都含有read()方法，用于读单个字节或者字节数组
    • 任何自OutputStream 或 Writer派生而来的类都含有write()方法，用于写单个字节或者字节数组

    # InputStream类型

    InputStream的作用是用来表示那些从不同数据源产生输入的类

    # 常见数据源类型

    1. 字节数组
    2. String对象
    3. 文件
    4. 管道
    5. 一个由其他种类的流组成的序列，可以将它们收集合并到一个流内
    6. 其他数据源，如网络连接等

    # 常见InputStream子类

    1. ByteArrayInputStream：允许将内存的缓冲区当做InputStream使用
    2. StringBufferInputStream（已弃用）：将String对象转换成InputStream
    3. FileInputStream：用于从文件中读取信息
    4. PipedInputStream：管道化。多线程使用
    5. SequenceInputStream ：将多个InputStream对象转换成单一InputStream
    6. FilterInputStream：抽象类，作为"装饰器"接口，在其内部修改InputStream行为方式

    # OutputStream类型

    Output的作用是用来表示那些将数据向目标输出的类

    # 常见目标类型

    1. 字节数组
    2. 文件
    3. 管道

    # 常见OutputStream子类

    1. ByteArrayOutputStream：在内存中创建缓冲区。所有输出流的数据都放置在此缓冲区
    2. FileOutputStream：用于将数据写入文件
    3. PipedOutputStream：管道化。多线程使用
    4. FilterOutPutStream：抽象类，作为"装饰器"接口，在其内部改变OutputStream行为方式

    # 字节流装饰器

    FilterInputStream和FilterOutputStream 是用来提供装饰器类接口以控制特定输入流（InputStream）和输出流（OutputStream）的两个类，分别是InputStream和OutputStream的子类，是装饰器的必要条件（以便能为所有正在装饰的对象提供通用接口）

    # 通过FilterInputStream从InputStream读取数据

    1. DataInputStream
    2. BufferedInputStream

    # 通过FilterOutputStream向OutputStream写入数据

    1. DataOutputStream
    2. PrintStream
    3. BufferedOutputStream

    # Reader和Writer

    # 概念

    • InputStream和OutputStream是面向字节形式的I/O
    • Reader和Writer是面向字符形式的I/O
    • 使用适配器(adapter)将字节I/O类转成字符I/O
    • InputStreamReader可以把InputStream转换为Reader
    • OutputStreamReader可以把OutputStream转换为Writer

    # 字节流和字符流的对应关系

    字节流 字符流
    InputStream Reader（适配器InputStreamReader）
    OutputStream Writer（适配器OutputStreamWriter）
    FileInputStream FileReader
    FileOutputStream FileWriter
    StringBufferInputStream（已弃用） StringReader
    （无对应的类） StringWriter
    ByteArrayInputStream CharArrayReader
    ByteArrayOutputStream CharArrayWriter
    PipedInputSream（多线程使用） PipedReader（多线程使用）
    PipedOutputStream（多线程使用） PipedWriter（多线程使用）

    # 字符流装饰器

    ✨对于InputStream和OutputStream来说，可以使用FilterInputStream和FilterOutputStream的装饰器子类来修改"流"已满足特殊需求。Reader和Writer的继承类层次结构继续沿用相同的" 装饰器"思想，分别使用的是FilterReader子类和FilterWriter子类。✨** 注意，尽管BufferedOutputStream是FilterOutputStream的子类，但是BufferedWriter并不是FilterWriter的子类 ** （尽管FIlterWriter是抽象类，没有任何子类，把它放在那里也只是把它作为一个占位符）

    字节流过滤器 字符流过滤器
    FilterInputStream FilterReader
    FilterOutputStream FilterWriter（抽象类，没有子类）
    BufferedInputStream BufferedReader（与BufferedInputStream继承结构不同）
    BufferedOutputStream BufferedWriter（**与BufferedOutputStream继承结构不同 **）
    PrintStream PrintWriter

    # 随机读写类RandomAccessFile

    RandomAccessFile不是InputStream和OutputStream继承层次结构的一部分，它只实现了DataInput和DataOutput 接口，它是一个完全独立的类。之所以这么做，是因为RandomAccessFile拥有和别的I/O类型本质不同的行为，使用它可以在一个文件内向前和向后移动

    # 文件访问模式

    • 'r'：只读
    • 'rw'：读写。如果文件不存在，则尝试创建该文件
    • 'rws'：读写，与'rw'类似。每次对文件内容或元数据的更新都会同步写入到磁盘
    • 'rwd'：读写，与'rw'类型。每次对文件内容的更新都会同步写入到磁盘，但不修改文件元数据

    # 标准I/O

    在许多操作系统中，程序的所有输入都可以来自标准输入，它的所有输出也都可以发送到标准输出，以及所有的错误信息都可以发送到标准错误

    # 概念

    1. 按照标准I/O模型，Java提供了System.in、System.out和System.err
    2. 标准I/O重定向，使用System的静态方法setIn()、setOut()和setErr()可以将标准输入、输出和错误I/O流进行重定向（I/O重定向操作的是字节流，而不是字符流）

    # 代码验证

    	// 1.将System.in转换成Reader 
	// System.in(InputStream) -> InputStreamReader -> 转换成Reader
    public static void reader() throws IOException {
        BufferedReader reader = new BufferedReader(new InputStreamReader(System.in));
        String line = "";
        while ((line = reader.readLine()) != null) {
            System.out.println(line);
            if ("exit".equals(line)) {
                break;
            }
        }
        reader.close();
    }

    // 2.将System.out转换成PrintWriter
	// System.out(PrintStream->OutputStream) -> PrintWriter
    public static void writer() {
        PrintWriter writer = new PrintWriter(System.out);
        writer.write("hello world");
        writer.flush(); // new PrintWriter(System.out,true);
        writer.close();
    }

	// 3.标准I/O重定向
	// System.setIn()
    // System.setOut()
    // System.setErr()
    public static void redirect() throws IOException {
        String inPath = "in.txt";
        String outPath = "out.txt";
        PrintStream console = System.out;
        BufferedInputStream in = new BufferedInputStream(new FileInputStream(inPath));
        PrintStream out = new PrintStream(new FileOutputStream(outPath));
      	// 重定向
        System.setIn(in);
        System.setOut(out);
        System.setErr(out);
      
        BufferedReader reader = new BufferedReader(new InputStreamReader(System.in));
        String line = "";
        while ((line = reader.readLine()) != null) {
            System.out.println(line);
            console.println(line);
        }
        out.close();
        System.setOut(console);

    }

    # 新I/O

    NIO提升了文件I/O和网络I/O的读取速度，之所以提升是因为它所使用的结构更接近于操作系统执行I/O的方式：通道和缓冲器，我们并不会直接和通道交互，只会和缓冲器进行交互，并把缓冲器派送到通道

    # 特性

    1. 通道要么从缓冲器获得数据，要么向缓冲器发送数据
    2. ByteBuffer是唯一直接与通道交互的缓冲器，其他视图缓冲器的底层实际也是ByteBuffer

    # FileChannel

    1. FileInputStream.getChannel()：产生可写的通道
    2. FileOutputStream.getChannel()：产生可读的通道
    3. RandomAccessFile.getChannel()：产生可读的、可写的通道

    # 转换数据

    缓冲器存储的是普通字节，为了把它们转换成字符，我们要么在输入它们的时候对其进行编码，这样的话，它们输出时才具有意义；要么在将其从缓冲器输出时对他们进行解码

    # 代码验证

    public static void 转换数据() throws IOException {
        FileChannel fc = new FileOutputStream("content.txt").getChannel();
        fc.write(ByteBuffer.wrap("hello world".getBytes()));
        fc.close();
        fc = new FileInputStream("content.txt").getChannel();
        ByteBuffer buffer = ByteBuffer.allocate(32);
        fc.read(buffer);
        buffer.flip();
        // 1. 输入: 未编码, 输出: 使用CharBuffer.toString()
        System.out.println("CharBuffer.toString() 未编码: " + buffer.asCharBuffer());
        buffer.rewind();
        // 2. 手动的将byte[]解码
        System.out.println("Decoded UTF-8: " + new String(buffer.array()));
        System.out.println("Decoded UTF-8: " + StandardCharsets.UTF_8.decode(buffer));
        // 3. 输入: 编码, 输出: 使用CharBuffer.toString()
        fc = new FileOutputStream("content.txt").getChannel();
        fc.write(ByteBuffer.wrap("hello world".getBytes(StandardCharsets.UTF_16BE)));
        fc.close();
        fc = new FileInputStream("content.txt").getChannel();
        buffer.clear();
        fc.read(buffer);
        buffer.flip();
        System.out.println("CharBuffer.toString() 已编码: " + buffer.asCharBuffer());
        // 4. 输入: 使用CharBuffer, 输出: 使用CharBuffer.toString()
        fc = new FileOutputStream("content.txt").getChannel();
        buffer = ByteBuffer.allocate(32);
        buffer.asCharBuffer().put("hello world");
        fc.write(buffer);
        fc.close();
        fc = new FileInputStream("content.txt").getChannel();
        buffer.clear();
        fc.read(buffer);
        buffer.flip();
        System.out.println("CharBuffer编码/解码: " + buffer.asCharBuffer());
    }

    # 缓冲器细节✨

    ByteBuffer是将数据移进移出通道的唯一方式。它由数据和可以高效访问及操作这些数据的四个索引组成，这四个索引是：mark（标记），position（位置），limit（界限），** capation（容量）**

    # 概念

    1. mark：可以将mark设置为当前position的值
    2. capacity：在读/写模式下都是固定的，就是我们分配的缓冲大小
    3. position：类似于读/写指针，表示当前读(写)到什么位置
    4. limit：
      1. 在写模式下表示最多能写入多少数据，此时和capacity相同
      2. 在读模式下表示最多能读多少数据，此时和缓存中的实际数据大小相同

    # 常用方法

    方法名称 功能描述
    capacity() 返回缓冲区容量
    clear() 清空缓冲区，将position设置为0，limit设置为容量
    flip() 将limit设置为position，position设置为0。用于准备从缓冲区读取写入的数据
    limit() 返回limit值
    limit(int lim) 设置limit值
    mark() 将mark设置为position
    position() 返回position值
    position(int pos) 设置position值
    remaining() 返回（limit - position）
    hasRemaining() 若有介于position和limit之间的元素，则返回true

    # 内存映射文件

    # 概念

    内存映射文件允许我们创建和修改那些因为太大而不能放入内存的文件

    # MappedByteBuffer

    MappedByteBuffer是一种特殊类型的直接缓冲器，使用时必须指定映射文件的初始位置和映射区域的长度。继承自ByteBuffer，它具有ByteBuffer的所有方法。使用"映射文件访问"效率更高

    1. 映射写：映射文件中所有输出必须使用RandomAccessFile，不能使用FileOutputStream
    2. 映射读：可以使用FileInputStream和RandomAccessFile

    # 代码验证

    public static void mapped() throws IOException {
        String filePath = "io.txt";
        // 映射读
        MappedByteBuffer readMapped = new FileInputStream(filePath).getChannel()
                .map(FileChannel.MapMode.READ_ONLY, 0, 8);
        while (readMapped.hasRemaining()) {
            System.out.println((char)readMapped.get());
        }
        // 映射写
        // 1. 使用FileOutputStream报错, NonReadableChannelException
        // MappedByteBuffer writeMapped = new FileOutputStream("content.txt").getChannel()
        //        .map(FileChannel.MapMode.READ_WRITE, 0, 12);
        // 2. 映射写只能使用RandomAccessFile
        FileChannel fc = new RandomAccessFile(filePath, "rw").getChannel();
        MappedByteBuffer writeMapped = fc.map(FileChannel.MapMode.READ_WRITE, 0, 8);
        writeMapped.put("admin".getBytes());
        fc.close();
    }

    # 文件加锁

    文件加锁机制，它允许我们同步访问某个作为共享资源的文件。Java的文件加锁机制直接映射到了本地操作系统的加锁工具

    # 怎么用

    通过FileChannel调用tryLock()或lock() ，就可以获得整个文件的FileLock。（SocketChannel、DatagramChannel和ServerSocketChannel不需要加锁，因为它们是从单进程实体继承而来的，通常不会在两个进程之间共享网络socket）

    1. tryLock()是非阻塞式的
    2. lock()是阻塞式的
    3. FileLock.release()可以释放锁
    4. 默认是独占锁，FileChannel.tryLock(pos，size，shared)，第三个参数指定是否是共享锁

    # 代码验证

    public static void main(String[] args) throws IOException {
        String filePath = "io.txt";
        FileChannel fc = new RandomAccessFile(filePath, "rw").getChannel();
  		// 尝试获取锁
        FileLock lock = fc.tryLock();
        try {
            ByteBuffer buffer = ByteBuffer.allocate(16);
            buffer.asCharBuffer().put("hello");
            TimeUnit.SECONDS.sleep(3);
            fc.write(buffer);
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
        // 释放锁
            lock.release();
        }
}

    # 压缩

    Java I/O类库中的类支持读写压缩格式的数据流，这些类属于InputStream和OutputStream的子类，这样做是因为压缩类库是按字节方式而不是字符方式处理的

    # 常见压缩类

    压缩类 功能
    CheckedInputStream GetCheckSum()为任何InputStream产生校验和解压缩
    CheckedOutputStream GetCheckSum()为任何OutputStream产生校验和压缩
    ZipInputStream 用于解压缩Zip文件格式的数据
    ZipOutputStream 用于将数据压缩成Zip文件格式
    GZIPInputStream 用于解压缩GZIP文件格式的数据
    GZIPOutputStream 用于将数据压缩成GZIP文件格式
    InflaterInputStream 解压缩的基类
    DeflaterOutputStream 压缩类的基类

    # 对象序列化

    # 概念

    • 序列化：将对象写入到IO流中
    • 反序列化：从IO流中恢复对象
    • 该类必须实现 java.io.Serializable 接口
    • 意义：序列化机制允许将实现序列化的Java对象转换位字节序列，这些字节序列可以保存在磁盘上，或通过网络传输，以达到以后恢复成原来的对象。序列化机制使得对象可以脱离程序的运行而独立存在
    • 使用场景：所有可在网络上传输的对象都必须是可序列化的，比如RMI（remote method invoke,即远程方法调用），传入的参数或返回的对象都是可序列化的，否则会出错； 所有需要保存到磁盘的java对象都必须是可序列化的。通常建议：程序创建的每个JavaBean类都实现Serializeable接口

    # 序列化的控制 Externalizable

    Externalizable接口扩展自java.io.Serializable接口。实现java.io.Serializable即可获得对类的对象的序列化功能。而Externalizable可以通过writeExternal() 和readExternal()方法可以指定序列化哪些属性

    # Serializable与Externalizable的区别

    1. Externalizable自定义序列化可以控制序列化的过程和决定哪些属性不被序列化

    2. Serializable序列化时不会调用默认的构造方法，而Externalizable序列化时会调用默认的构造方法

    3. 使用Externalizable时，必须按照写入时的确切顺序读取所有字段状态。否则会产生EOFException

      异常。而Serializable接口没有这个要求

    # 代码验证

    public static void main(String[] args) throws Exception {
        Blip1 blip1 = new Blip1();// 构造方法是public
        Blip2 blip2 = new Blip2();// 构造方式是private
        // 序列化
        ByteArrayOutputStream byteArrayOutputStream = new ByteArrayOutputStream();
        ObjectOutputStream outputStream = new ObjectOutputStream(byteArrayOutputStream);
        outputStream.writeObject(blip1);
        outputStream.writeObject(blip2);

        // 反序列化
        ByteArrayInputStream byteArrayInputStream = new ByteArrayInputStream(byteArrayOutputStream.toByteArray());
        ObjectInputStream inputStream = new ObjectInputStream(byteArrayInputStream);
        Blip1 b1 = (Blip1) inputStream.readObject();
        Blip2 b2 = (Blip2) inputStream.readObject();

    }

    static class Blip1 implements Externalizable {
        public Blip1() {
            System.out.println("Blip1#construct()");
        }
        @Override
        public void writeExternal(ObjectOutput out) throws IOException {
            System.out.println("Blip1#writeExternal()");
        }
        @Override
        public void readExternal(ObjectInput in) throws IOException, ClassNotFoundException {
            System.out.println("Blip1#readExternal()");
        }
    }

    static class Blip2 implements Externalizable {
        private Blip2() {
            System.out.println("Blip2#construct()");
        }

            System.out.println("Blip2#writeExternal()");
        }

            System.out.println("Blip2#readExternal()");
        }
    }
