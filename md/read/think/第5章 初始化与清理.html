    第5章 初始化与清理

    qinhaoqvq

    # 构造器确保初始化

    1. Java中，构造器的名称必须与类名完全相同
    2. 不包含任何参数的构造器叫做默认构造器，通常也称为无参构造器。多个构造函数是以重载的形式存在
    3. 构造器是一种特殊类型的方法，因为它没有返回值。但是它与返回值为空（void）的方法也不是同一概念
    4. 如果你写的类中没有构造器，则编译器会自动帮你创建一个无参构造器
    5. ✨构造器可以看做是静态方法

    # 方法重载

    它是指我们可以在同一个类中定义一些名称相同的方法，通过定义不同的参数列表（包括参数类型、参数个数和参数顺序）来区分这些方法（返回值不能作为重载的判断依据），这种现象就是方法重载

    # 涉及基本类型的重载

    # 实参类型小于形参类型

    如果实际传入参数的数据类型小于方法中声明的形参类型，实际数据的数据类型就会被提升。char类型比较特殊，如果无法找到接收char类型参数的方法，就会把char直接提升为int类型

    # 实参类型大于形参类型

    如果实际传入参数的数据类型大于方法中声明的形参类型，就需要通过强制类型转换，将实际数据的数据类型转换为较小的类型。如果不进行强制类型转换，编译器将直接报错

    # 代码验证

    public class PrimitiveOverloading {

    public static void main(String[] args) {
      
        // 1. 实际传入参数类型小于重载方法声明的形参类型
        // 隐式类型转换 int -> long
        m1(5);
        // char类型比较特殊, char -> int -> long
        m1('a');

        // 2. 实际传入参数类型大于重载方法声明的形参类型
        m1((long) 10.00);
    }

    public static void m1(long x) {
        System.out.println("m1(long): " + x);
    }
}

    # 方法覆盖

    它是指我们可以在子类中定义与父类中方法名字、参数列表、以及返回值都相同的方法。

    • 覆盖不可降低方法的访问权限
    • 覆盖不可提升方法抛出异常类型的范围
    • 协变返回类型。它表示在子类中的被覆盖方法可以返回父类方法的返回值类型的子类

    # this关键字

    this通常都是指"这个对象"或者"当前对象"，而且它本身表示对当前对象的引用

    # static关键字

    static主要解决了为某特定域分配单一存储空间，而不去考虑要创建多少对象，甚至根本不需要创建任何对象的情况，或者定义某个方法不与包含它的类的任何对象关联在一起的情况。因此便有了static域和static方法，也被称为类数据和类方法。

    1. 在static方法的内部不能调用非静态方法，反过来是可行的
    2. 可以在没有创建任何对象的前提下，通过类本身来调用static方法

    # ✨初始化

    # 初始化类型

    • 分配给对象的存储空间进行初始化，值为二进制的零（参考多态）
    • 成员初始化
    • 构造方法初始化
    • 静态数据初始化
    • 显示的静态初始化（静态代码块static{}）
    • 非静态实例初始化（非静态代码块{}）

    # 初始化顺序

    序号 初始化
    1 父类静态变量/静态代码块
    2 子类静态变量/静态代码块
    3 父类成员变量/非静态代码块
    4 父类构造方法
    5 子类成员变量/非静态代码块
    6 子类构造方法
    • 子类的静态变量和静态代码块的初始化是在父类的变量、非静态代码块和构造器初始化之前就完成的
    • 静态变量、静态代码块顺序取决于它们在类中出现的先后顺序
    • 变量、非静态代码块初始化顺序取决于它们在类中出现的先后顺序

    # 代码验证

    // 1. 定义父类
public class InitializationFather {
    // 成员变量
    private InitializationEntity entity = new InitializationEntity("father成员变量");
    // 静态变量
    private static InitializationEntity staticEntity = new InitializationEntity("father静态变量");
    // 静态代码块
    static {
        System.out.println("father: static代码块");
    }
    // 非静态代码块
    {
        System.out.println("father: 代码块");
    }
    // 构造方法
    public InitializationFather() {
        System.out.println("father: 构造器");
    }
}

// 2. 定义子类
public class InitializationChild extends InitializationFather {
    // 1. 成员变量
    private InitializationEntity entity = new InitializationEntity("child成员变量");
    // 2. 静态变量
    private static InitializationEntity staticEntity = new InitializationEntity("child静态变量");

    // 3. 静态代码块
    static {
        System.out.println("child: static代码块");
    }

    //4. 非静态代码块
    {
        System.out.println("child: 代码块");
    }
    //5. 构造方法
    public InitializationChild() {
        System.out.println("child: 构造方法");
    }

    public static void main(String[] args) {
        InitializationChild child = new InitializationChild();
    }
}

// 3. 变量对象
public class InitializationEntity {

    static {
        System.out.println("entity: 静态代码块");
    }

    public InitializationEntity(String from) {
        System.out.println("entity: 构造方法, from: " + from);
    }
}
