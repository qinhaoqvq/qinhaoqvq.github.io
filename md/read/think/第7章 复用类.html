    第7章 复用类

    qinhaoqvq

    # 介绍

    复用类，说白了就是重复使用类，此方法的窍门在于使用类而不破坏现有程序代码。在Java中复用的三种常用方式是：组合、继承和代理

    # 组合、继承和代理

    # 组合

    我们可以在一个新类的定义中引用其他对象，这就是组合

    • 使用基本数据类型时会被初始化成二进制的零
    • 使用引用数据类型时会被初始化为null

    # 继承

    使用extends关键在基类的基础上创建新类，这就是继承

    • 继承能够访问基类中非private的变量和方法（如果基类与新类不在同一包下，那也不能访问default的变量和方法）
    • 初始化基类
      • 当创建了一个子类的对象时，该对象也包含了一个基类的子对象
      • 在构造方法中调用基类的构造方法来执行初始化
    • super能够显示的调用基类的构造方法
    • ✨只有非private方法才能被覆盖

    # 代理

    我们将一个成员对象置于所要创建的新类中（就像组合），但与此同时我们在新类中暴露了该成员对象的所有方法（就像继承），这就是代理

    # 组合和继承对比

    • 组合和继承都是允许在新的类中放置子对象，组合是显式地这样做，而继承则是隐式地这样做
    • 组合可以将引用对象隐藏起来，在外部看来是在使用新类定义的接口，在内部其实所有功能都由引用对象实现
    • 组合可以将引用对象暴露出来，允许用户直接访问
    • ✨该用组合还是继承？判断是否需要从新类向基类进行向上转型，如果必须向上转型，则继承是必要的

    # ✨类型转换

    # 宽转型、窄转型

    宽转型和窄转型是针对基本数据类型而言的，窄转型是不安全的要进行强制类型转换，可能发生精度丢失问题

    # 向上转型、向下转型

    向上转型和向下转型是针对对象而言的，它是专用类型和通用类型之间的相互转换。

    该术语的使用是有其历史原因的，并且是以类继承图的绘制方法为基础的。

    • 向上转型 AKA 自动转换
      • 向上转型是安全的，因为基类不会具有超出子类接口的范围
      • 向上转型会丢失具体的类型信息（算是缺点）
      • ✨基类引用指向子类对象由于向上转型了，它只能访问基类中拥有的方法和属性，而对于子类中存在而基类中不存在的方法，该引用是不能使用的，尽管是重载该方法（算是缺点）
      • ✨若子类重写了基类类中的某些方法，在调用该些方法的时候，必定是使用子类中定义的这些方法（被称后期绑定或者动态绑定）
    • 向下转型 AKA 强制转换
      • 向下转型是不安全的，不正确的向下转型会抛出ClassCastException
    • 运行时类型识别（RTTI）
      • 在运行期间对类型进行检查的行为就被称作RTTI

    # 代码验证

    // 父类
public class Instrument {
    public void play() {
        System.out.println("Instrument#play()方法被调用");
    }
    public void overload() {
        System.out.println("Instrument#overload()方法被调用");
    }
    public void override() {
        System.out.println("Instrument#override()方法被调用");
    }
}
// 子类
public class Wind extends Instrument {
    public static void main(String[] args) {
        // 创建指向子类的父类引用, 即, 向上转型
        Instrument instrument = new Wind();
        // 1. 子类覆盖了父类中的方法, 在实际执行方法时, 调用的是子类的方法
        // 输出结果: Wind#override()方法被调用
        instrument.override();
        // 2. Error, 指向子类的父类引用, 不能访问父类中不存在的方法, 即使是重载方法
        // instrument.overload(1);
    }
    public void overload(int i) {
        System.out.println("Wind#overload()方法被调用");
    }
    @Override
    public void override() {
        System.out.println("Wind#override()方法被调用");
    }
}

    # final关键字

    # final数据

    • final基本数据类型，数值恒定不变
    • final引用数据类型，引用恒定不变
    • 必须在域的定义处或者每个构造方法中对final变量进行赋值

    # final方法

    ✨混淆问题：

    假如，在基类中声明一个private hello()的方法，在子类中试图定义一个同名的hello方法来覆盖，编译器也不会有错误信息。但其实是无效的，因为覆盖 只有在某方法是基类的接口的一部分时才会出现。即，必须能够将一个对象向上转型为它的基本类型并调用相同的方法。如果某方法为private，那么它就不是基类的接口的一部分。

    • final方法可以把方法锁定，防止任何继承类修改
    • 编译器会将final方法都转为内嵌调用，消除方法调用的开销。现在已经过时，虚拟机会自动优化
    • 类中所有的private方法都隐式的指定为是final的，因为private方法无法使用，所以也就无法覆盖它

    # final类

    当将某个类的整体定义为final时，那么该类将无法被继承
