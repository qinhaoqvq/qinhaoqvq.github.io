    第15章 泛型

    qinhaoqvq

    # 概念

    # 是什么

    泛型即参数化类型，可以使代码能够用于"某种不具体的类型"，而不是一个具体的接口或类

    # 能干嘛

    泛型的主要目的之一就是用来指定容器要持有什么类型的对象，而且由编译器来保证类型的正确性

    # 特性

    • 泛型不支持协变
    • 泛型的类型参数只能是类类型（包括自定义类），不能是简单类型
    • 由于类型擦除的原因，f(List<A> l)与f(List<B> l)是同样类型，编译器没办法根据参数区别重载方法

    # 泛型的使用

    泛型有三种使用方式，分别为：泛型类、泛型接口、泛型方法。

    # 泛型类

    泛型类型用于类的定义中，被称为泛型类。通过泛型可以完成对一组类的操作对外开放相同的接口。最典型的就是各种容器类，如：List、Set、Map

    class 类名称 <泛型标识：可以随便写任意标识号，标识指定的泛型的类型>{
  private 泛型标识 /*（成员变量类型）*/ var; 
  ...
}

    # 泛型接口

    泛型接口与泛型类的定义及使用基本相同。未传入泛型实参时，与泛型类的定义相同，在声明类的时候，需将泛型的声明也一起加到类中

    class FruitGenerator<T> implements Generator<T>{
    @Override
    public T next() {
        return null;
    }
}

    # 泛型方法

    # 定义

    1. ✨要将泛型参数列表置于返回值前，可以理解为是声明此方法为泛型方法
    2. 只有声明了的方法才是泛型方法，泛型类中使用了泛型成员的方法不是泛型方法
    3. 表明该方法将使用泛型类型T，此时在方法中可以使用泛型类型T

    # 使用

    # 简单的泛型方法
    public class GenericMethod {
    // 
    public <T> void same(T obj) {
        System.out.println(obj.getClass());
    }

    public static void main(String[] args) {
        GenericMethod me = new GenericMethod();
        me.same("a");
        me.diff(100);
      	me.diff(10L);
      	me.diff(10F);
    }
}
    # 泛型类中的泛型方法

    1.由于泛型方法在声明的时候会声明泛型，因此即使在泛型类中并未声明泛型，编译器也能够正确识别泛型方法中识别的泛型

    2.可以类型与T相同，也可以不同，如果使用泛型T，注意这个T是一种全新的类型，可以与泛型类中声明的T不是同一种类型

    public class GenericMethod<T> {
    private T member;
  
	// 由于泛型方法在声明的时候会声明泛型<E>，因此即使在泛型类中并未声明泛型，编译器也能够正确识别泛型方法中识别的泛型。
  	// 可以类型与T相同，也可以不同，如果使用泛型T，注意这个T是一种全新的类型，可以与泛型类中声明的T不是同一种类型
    public <E> void diff(E obj) {
        System.out.println(obj.getClass());
    }

    public static void main(String[] args) {
        GenericMethod<Integer> me = new GenericMethod<>();
        me.same("a");
        me.diff(100);
    }
}
    # 静态方法与泛型

    ✨对于一个static方法而言，无法访问泛型类的类型参数，所有，如果static方法需要使用泛型能力，就必须是其称为泛型方法。即，如果静态方法要使用泛型的话，必须将静态方法也定义成泛型方法

    public class StaticGenericMethod<T> {

    public void show1(T obj) {
        System.out.println(obj.getClass());
    }

    // 如果在类中定义使用泛型的静态方法，需要添加额外的泛型声明（将这个方法定义成泛型方法），即使静态方法要使用泛型类中已经声明过的泛型也不可以
    // 编译报错: cannot be referenced from a static context 无法从静态上下文中引用非静态 类型变量 T
    public static void show2_1(T obj) {
    }
	// 正确使用
    public static <E> void show2_2(E obj) {
        System.out.println(obj.getClass());
    }
}

    # 类型参数推断✨

    类型参数推断（type argument inference）：当使用泛型类的时候，必须在创建对象的时候指定类型参数的值，但是使用泛型方法的时候，通常不必指明参数类型，因为编译器会为我们找出具体类型

    # 类型擦除

    # 是什么

    1. Java泛型是使用擦除来实现的，这意味着当你在使用泛型时，其泛型信息只存在于编译期，而在运行期任何具体的类型信息都被擦除，你唯一知道的就是你在使用一个Object对象
    2. Java的泛型基本上都是在编译器这个层次上实现的，在运行期生成的字节码中是不包含泛型中的类型信息的，使用泛型的时候加上类型参数，在编译器编译的时候会去掉

    # 特性

    1. 在泛型代码内部，无法获得任何有关泛型参数类型的信息
    2. 类型擦除将泛型类型参数擦除到它的第一个边界，编译器实际会把类型参数替换成它的边界类型

    # 擦除的问题

    泛型不能用于显式地引用运行时类型的操作之中，例如类型转换、instanceof操作和new表达式，因为此时所有关于参数的类型信息都丢失了（或者擦除到边界）

    # 擦除的补偿

    // 缺陷：任何在运行时需要知道确切类型信息的操作都无法完成
class Erasure<T> {
    public static void f(Object o) {
        if(o instanceof T) { //Error
            T var = new T(); //Error
            T[] array = new T[10]; //Error
            T[] array1 = (T[])new Object[10]; //Unchecked warning
        }
    }
}
    • obj instanceof T进行补偿
      • 使用动态的isInstance()
    • 创建类型实例
      • 传递一个Class工厂对象，使用Class.newInstance()来创建这个类型的对象
    • 创建泛型数组
      • 成功创建泛型数组的唯一方式：创建一个被类型擦除的数组，然后对其转型

    # 边界

    边界使得你可以在用于泛型的参数类型上设置限制条件，<T extends Object>Object以及Object的子类可以传递给类型参数T，用于定义泛型方法、泛型类。

    # 代码验证

    public class Erasure<T extends Erasure.Pet> {
    public T member;
  
    public static void main(String[] args) {
        // List表示持有任何Object类型的原生List
        // List<?>表示具有某种特定类型的非原生List,只不过不知道是什么类型
        List list1 = new ArrayList();
        List<Pet> list2 = new ArrayList<>();
        System.out.println(list1.getClass());
        System.out.println(list2.getClass());
		
        Erasure<Pet> e = new Erasure<>();
        Field[] fields = e.getClass().getDeclaredFields();
        System.out.println(e.getClass());
        System.out.println(fields[0].getName() + "\t" + fields[0].getType());

    }
    static class Pet {
    }
    static class Dog extends Pet {
    }
}

    # 总结

    • 因为擦除移除了类型信息，所以，可以用无界泛型参数调用的方法只有Object的方法

    • ✨在虚拟机中，Erasure被擦除成Erasure（原生类型），Erasure类内部的T被擦除到上边界

    # 通配符"?"

    List<Fruit> flist = new ArrayList<Apple>()这样的语句是无法通过编译的，尽管 Apple 是 Fruit 的子类。之所以编译报错，是因为泛型与数组不同，泛型没有内建的协变类型。在使用泛型时，类型信息在编译过后就被类型擦除了，所以编译器和运行时系统无法进行类型检查。**那么如果我们确实需要建立这种 "向上转型" 的关系怎么办呢？这正是通配符所允许的。**✨

    # 无界通配符

    无界通配符代表某个具体的类型，而原生类型代表任何类型

    1. 原型类型什么类型都能放，可以取值，只能用Object
    2. 使用了无界通配符后，什么类型都不能放（除了null值），可以取值，但是不知道确切的类型，只能用Object

    # 代码验证

    public static void main(String[] args) {
  		// 原生类型
  		List list = new ArrayList();
  		list.add(null);
  		list.add(1);
 		// 无界通配符<?> 
        List<?> list = new ArrayList<>();
        list.add(null);
        //list.add(1); compiler error
}

    # 上边界通配符（泛型协变）

    • List<? extends Fruit> 它表示某种特定类型 ( Fruit 或者其子类 ) 的 List，但是并不关心这个实际的类型到底是什么。所以，也就无法知道List到底持有什么类型的对象，那么也无法向其添加对象
    • 但是获取某个 Fruit 对象的方法是安全的。因为我们知道，在这个 List 中，不管它实际的类型到底是什么，但肯定能向上转型为 Fruit，所以编译器允许返回 Fruit

    # 下边界通配符（泛型逆变）

    • List<? super Fruit>，它表示某种类型的 List，这个类型是 Fruit 或者其父类型，到底是什么类型无法确定。但肯定是 Fruit 或其父类型，所以，可以向这个 List 添加Fruit 或者其子类型的对象，因为这些对象都可以向上转型为Fruit
    • 但是获取某个 Fruit 对象的方法是不安全的，因为我们不知道，在这个 List 中，它实际的类型到底是什么，所以只能获取到Object的对象

    # 上下边界通配符的总结✨

    • 对于<?>，无法向里面添加元素（除了null），可以读取其中的元素，但读取出来的元素会变为 Object 类型
    • 对于<? extends T>，无法向里面添加元素（除了null），只能读取其中的元素
    • 对于<? super T>，可以读取其中的元素，但读取出来的元素会变为 Object 类型

    # 代码验证

    public static void main(String[] args) {
        // 1. 编译期,未知类型,不接受任何类型数据,但是可以加入Null值
        List<?> list1 = new ArrayList<>();
        // list1.add("a"); Compiler Error
        // list1.add(100); Compiler Error
        list1.add(null);
  		Object object = list1.get(0);

        // 2. 无法向里面添加元素（除了null），只能读取其中的元素
        List<? extends Pet> list2 = new ArrayList<>();
        // list2.add(new Pet()); Compiler Error
        // list2.add(new Dog()); Compiler Error
        Pet pet = list2.get(0);


        // 3. 可以读取其中的元素，但读取出来的元素会变为 Object 类型
        List<? super Pet> list3 = new ArrayList<>();
        Object object = list3.get(0);
        list3.add(new Pet());
        list3.add(new Dog());
    }

    # 详细解释✨

    # 前提

    基于List来理解通配符的实际意义

    1. 不允许的写操作有add函数，因为它的函数签名是boolean add(E e)，此时这个形参E就变成了一个涉及了通配符的类型
    2. 而不允许的读操作有get函数，因为它的函数签名是E get(int index)，此时这个返回值E就变成了一个涉及了通配符的类型

    # 总结

    1. 引用类型为List<? super Fruit>，那么合法行为只能是something extends Fruit类型赋值给? super Fruit，要做到这样的事，只能是方法的形参类型是类型参数E。因为这样使用者就可以把实参控制为something extends Fruit，传递给形参类型为? super Fruit的方法以实参（此时E被? super Fruit代替）。而形参类型为类型参数的方法，一般都是"写操作"。
    2. 引用类型为List<? extends Fruit>，那么合法行为只能是? extends Fruit类型赋值给something super Fruit，要做到这样的事，只能是方法的返回值类型是类型参数E。因为这样使用者就可以把函数返回赋值的引用控制为something super Apple，然后用这个引用来接返回值类型为? extends Fruit的返回值（此时E被? extends Fruit代替）。而返回值类型为类型参数的方法，一般都是"读操作"。

    # T和?的区别✨

    • T 是一个参数，也是一个确定的类型。通常用于泛型类和泛型方法的定义
    • ? 是一个不确定的类型，但终究代表一个具体类型。通常用于泛型方法的调用代码和形参，不能用于定义类和泛型方法

    # Class和Class<?>的区别

    • Class<T>在实例化的时候，T 要替换成具体类
    • Class<?>它是个通配泛型，? 可以代表任何类型，主要用于声明时的限制情况

    # 代码验证

    // 可以
public Class<?> clazz;
// 不可以，因为 T 需要指定类型
public Class<T> clazz;

    # 元组

    元组是将一组对象直接打包存储与其中的一个单一对象，这个容器对象允许读取其中元素，但是不允许向其中存放新的对象。通常元素可以具有任意长度，同时，元组中的对象可以是任意不同的类型

    public class TupleObject<A, B> {
    public final A a;
    public final B b;

    public TupleObject(A a, B b) {
        this.a = a;
        this.b = b;
    }
	// 定义使用
    public static void main(String[] args) {
        TupleObject<String, Integer> tuple = new TupleObject<String, Integer>("string", 0);
        System.out.println(tuple.a);
        System.out.println(tuple.b);
    }
}

    # 动态类型安全

    java.util.Collections提供了工具类，可以解决类型检查的问题。静态方法：checkedCollection()、checkedList()、checkedMap()、checkedSet()、checkedSortedMap()和checkedSortedSet()，这些方法都可以创建受检查的容器。如果试图插入类型不正确的对象时会抛出ClassCastException，这与原生类型的容器形成了对比。
