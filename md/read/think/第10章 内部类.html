    第10章 内部类

    qinhaoqvq

    # 是什么

    可以将一个类的定义放在另一个类的定义内部，这就是内部类

    # 能干嘛

    内部类可以继承多个具体的或抽象的类的能力，这使得多重继承 的解决方法变得完整。接口解决了部分问题，而内部类有效地实现了多重继承

    # 不同的内部类

    在类中定义的内部类，是可以在外部类中创建其对象的。但是，在方法和作用域中定义的内部类，一旦出了方法和作用域的范围，这个内部类便不可以再被访问

    # 类中的内部类

    • 在外部类的非静态方法之外的位置创建内部类对象，必须使用OuterClassName.InnerClassName 的方式指明这个对象的类型
    • 必须使用外部类的对象来创建内部类的对象（静态内部类不必需要）
      • ✨因为内部类的对象会拥有外部类对象的引用
    • 内部类可以访问外部类的所有成员
    • 内部类可以访问外部类对象的所有成员（通过OuterClassName.this获取外部类对象）
    • 内部类定义的类型（如变量）会屏蔽其作用域范围内的其他同名类型（变量）
    • 继承会把内部类一起继承，内部类就是外部类的一部分，属于它的成员

    # .this

    在内部类生成外部类对象的引用，OuterClassName.this，在内部类显示的使用外部类对象

    # .new

    创建某个对象内部类的对象时，需要使用该语法，OuterClassInstance.new InnerClassName()

    # 方法和作用域中的内部类

    方法和作用域中的内部类也是其一部分，离开了方法和作用域其内部类便不可访问

    # 匿名内部类

    匿名内部类不能访问定义在外部类未加final修饰的对象（JDK1.8之后不加final也可以访问）

    # 嵌套类（静态内部类）

    • 嵌套类与外部类对象之间无联系，即，嵌套类的对象不会拥有外部类对象的引用
    • 嵌套类不可以访问外部类的任何的非静态成员
    • 嵌套类可以定义static的变量和static的方法

    # 接口内部类

    如果想要创建某些公共的代码，使得它们可以被某个接口的所有不同实现所共用，那么就可以使用接口内部类（接口内部类都是public和static的）

    # 局部内部类

    # 定义

    在代码块中创建的没有访问修饰符的内部类，就是局部内部类

    # 特性

    局部内部类与匿名内部类具有相同的行为和能力，在代码块之外的位置是不可见的，都可以访问当前代码块内的常量（final修饰的对象）以及此外部类的所有成员

    # 能干嘛

    局部内部类有一个已命名的构造方法，支持重载构造方法或者创建多个该内部类的对象

    # 内部类的继承

    class WithInner {
    class Inner {
    }
}
public class InheritInner extends WithInner.Inner {
    // public InheritInner() 编译报错
    // 子类构造方法不仅需要传递外部类对象的引用, 而且还需要执行外部类对象.super()
    public InheritInner(WithInner wi) {
        wi.super();
    }
}

    # 内部类不可以被覆盖

    新建一个类继承了外部类A，并创建了与外部类A中的内部类A1同名的内部类，两个内部类无明显变化。这两个内部类是完全独立的两个实体，各自在自己的命名空间内

    # 内部类标识符

    由于每个类都会产生一个.class文件，其中包含了如何创建该类型的对象的全部信息（此信息生成一个Class对象），那么内部类也必须生成一个.class文件以包含它们的Class对象信息。这些类文件的命名是有规则的：OuterClassName$InnerClassName.class，如果匿名内部类，编译器会生成一个数字作为其标识符
